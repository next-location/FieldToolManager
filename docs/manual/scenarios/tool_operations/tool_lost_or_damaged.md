# 道具の紛失・破損対応マニュアル

## 1. 概要

道具の紛失や破損が発生した際の適切な対応手順を説明します。早期対応により、業務への影響を最小限に抑え、再発防止につなげることができます。

**このマニュアルで解決できること**:
- 道具が見つからないときの対応フロー
- 破損した道具の処理方法
- 修理に出す手順
- 道具の削除・廃棄手続き
- 再発防止のための記録管理

---

## 2. 紛失・破損の種類と対応方針

### 2-1. 紛失（Lost）

| 状況 | 対応レベル | 初動対応 |
|------|-----------|----------|
| **一時的に見つからない** | 🟡 注意 | 30分〜1時間探す |
| **現場で紛失** | 🟠 警戒 | 現場責任者に報告、全員で捜索 |
| **盗難の可能性** | 🔴 重大 | 即座に上司・管理者に報告 |

### 2-2. 破損（Damaged）

| 状況 | 対応レベル | 初動対応 |
|------|-----------|----------|
| **軽微な破損** | 🟢 軽微 | 自社で簡易修理可能 |
| **修理が必要** | 🟡 注意 | 修理業者に見積依頼 |
| **使用不能** | 🟠 警戒 | 即座に使用中止、代替品手配 |
| **安全上の危険** | 🔴 重大 | 即座に使用中止、廃棄検討 |

---

## 3. 道具の紛失時の対応フロー

### ステップ1: 初動対応（発見から30分以内）

#### 1-1. 自分で探す（5〜10分）

**確認場所**:
- ✅ 作業現場（最後に使った場所）
- ✅ 車両・トラックの荷台
- ✅ 休憩スペース
- ✅ 倉庫・工具箱
- ✅ 別の現場に持って行っていないか

#### 1-2. システムで最終位置を確認（5分）

1. スマホまたはPCで **ザイロク** にアクセス
2. 左サイドメニュー **「道具」** をクリック
3. 検索ボックスで道具名またはシリアル番号を入力
4. 道具詳細画面を開く
5. **「移動履歴」** タブをクリック
6. 最新の移動記録を確認

**確認項目**:
```
最終移動日時: 2026年1月20日 14:30
移動種別: 持ち出し
移動元: 倉庫
移動先: 現場A
実施者: 山田太郎
備考: 配管工事用
```

→ この情報から「現場Aに持ち出したはず」と分かります。

#### 1-3. チームメンバーに確認（10分）

**質問例**:
- 「このドリル（#001）見なかった？」
- 「昨日、現場Aで使ってたパイプレンチ、誰か知らない？」
- 「トラックに積んだままになってない？」

---

### ステップ2: 報告（30分〜1時間経過）

#### 2-1. 上司・現場責任者に報告

**報告内容**:
```
【紛失報告】
道具名: 電動ドリル
シリアル番号: #001
型番: マキタ DF330DWX
最終確認日時: 2026年1月20日 14:30
最終確認場所: 現場A（配管作業エリア）
捜索状況: 現場内を30分捜索したが見つからず
次のアクション: 全員で捜索、警察に届出検討
```

#### 2-2. システムに記録（任意）

道具詳細画面の **「備考」** 欄に一時的にメモを残すことができます（ただし、ステータスは変更しない）。

```
【備考例】
2026-01-20 15:00 - 現場Aで紛失の可能性あり。捜索中。（報告者: 山田）
```

---

### ステップ3: 捜索・確認（1時間〜1日）

#### 3-1. 組織的な捜索

- 🔍 現場全体を複数人で徹底捜索
- 🔍 車両・トラックを再確認
- 🔍 他の現場に間違って持って行っていないか確認
- 🔍 同じ日に作業した全員に聞き取り

#### 3-2. 近隣・警察への確認

- 📞 現場近隣の住民・店舗に落とし物として届いていないか確認
- 📞 警察署に遺失物届を提出
- 📞 他の業者が間違って持って行った可能性を確認

---

### ステップ4: システム上の処理（紛失確定後）

#### 4-1. 管理者に削除を依頼

**重要**: 道具の削除は **管理者（Admin）権限のみ** が実行できます。

**依頼方法**:
1. 上司または管理者に以下の情報を伝える

```
【道具削除依頼】
道具名: 電動ドリル
シリアル番号: #001
削除理由: 紛失（現場A、2026-01-20）
捜索結果: 3日間捜索したが発見できず
警察届出: 遺失物届提出済み（受理番号: XX-XXXX）
代替品手配: 既に手配済み（#015を新規登録）
```

#### 4-2. 管理者による削除手順

**管理者のみ実行可能**:

1. 左サイドメニュー **「道具」** をクリック
2. 削除する道具をクリック
3. 道具詳細画面で **「削除」** ボタンをクリック
4. 確認ダイアログが表示される
5. **「本当に削除しますか？」** → **「削除する」** をクリック
6. 削除が完了すると道具一覧から消えます

**注意**: 削除は **論理削除** です。データベースには残りますが、一覧には表示されなくなります。

#### 4-3. 削除後の確認

- ✅ 道具一覧画面で該当道具が表示されないことを確認
- ✅ 在庫数が正しく減っていることを確認
- ✅ 移動履歴は削除前のものが保持されます（監査用）

---

## 4. 道具の破損時の対応フロー

### ステップ1: 安全確認と使用中止

#### 4-1. 即座に使用を中止

**危険な状態の例**:
- ⚠️ 電動工具のコードが切れている
- ⚠️ ハシゴの脚が折れている
- ⚠️ 刃物が欠けている
- ⚠️ 安全装置が壊れている

**対応**:
1. **即座に使用中止**
2. 「故障中・使用禁止」の紙を貼る
3. 他の人が誤って使わないように隔離
4. 上司に報告

#### 4-2. 破損状況の記録

**写真撮影**:
- 📸 破損箇所の全体写真
- 📸 破損箇所のアップ写真
- 📸 型番・シリアル番号が分かる写真

**記録内容**:
```
【破損報告】
道具名: 電動ドリル
シリアル番号: #001
破損箇所: バッテリー接続部分
破損状況: 接続端子が折れて充電不可
破損原因: 落下（高さ約1.5m）
発生日時: 2026年1月20日 10:30
発生場所: 現場A（2階作業エリア）
発見者: 山田太郎
```

---

### ステップ2: 修理可否の判断

#### 2-1. 修理可能かチェック

| 破損内容 | 修理可否 | 対応 |
|---------|---------|------|
| **軽微な傷・汚れ** | ✅ 自社対応可能 | 清掃・簡易修理 |
| **部品交換で直る** | ✅ 修理推奨 | 修理業者に見積依頼 |
| **修理費が高額** | ⚠️ 要検討 | 修理費 vs 新品購入を比較 |
| **修理不能** | ❌ 廃棄 | 新品を購入 |

#### 2-2. 修理見積の取得

1. 修理業者に写真を送付
2. 見積を取得（修理費、納期）
3. 上司に報告・承認を得る

**判断基準（目安）**:
```
修理費が新品価格の50%以下 → 修理
修理費が新品価格の50%以上 → 新品購入を検討
```

---

### ステップ3: 修理に出す手順

#### 3-1. システム上で「修理中」に変更

1. 左サイドメニュー **「道具」** をクリック
2. 修理に出す道具をクリック
3. 道具詳細画面の **「移動登録」** ボタンをクリック
4. **移動先**: **「🔧 修理」** を選択
5. **備考**: 修理内容と業者名を記入

```
【備考例】
修理内容: バッテリー接続端子交換
修理業者: 株式会社○○工具サービス
見積金額: 8,000円
修理期間: 1週間（1/27返却予定）
```

6. **「登録する」** をクリック

#### 3-2. 実物を修理業者に送付

- 📦 破損した道具を梱包
- 📄 修理依頼書を添付
- 🚚 修理業者に発送または持ち込み

#### 3-3. 修理完了後の処理

**修理業者から返却されたら**:

1. 動作確認を実施
2. 左サイドメニュー **「道具」** をクリック
3. 修理完了した道具をクリック
4. **「移動登録」** ボタンをクリック
5. **移動先**: **「🏢 倉庫」** を選択
6. **備考**: 「修理完了（業者: ○○工具サービス、費用: 8,000円）」
7. **「登録する」** をクリック

---

### ステップ4: 廃棄・削除手順

#### 4-1. 廃棄が必要なケース

- ❌ 修理不能
- ❌ 修理費が新品価格を超える
- ❌ 安全上の問題で使用継続が危険
- ❌ 旧型で部品が入手できない

#### 4-2. 廃棄手順

1. 上司・管理者に廃棄申請
2. 承認後、実物を廃棄（産業廃棄物として適切に処理）
3. 管理者がシステムから削除（手順は「紛失時の削除手順」と同じ）

#### 4-3. 代替品の手配

- 🛒 同じ道具の新品を購入
- 📝 道具マスタに新しい個別アイテムを追加登録

## 5. 再発防止策

### 5-1. 紛失防止

**対策1: 道具セットで管理**

- よく使う道具の組み合わせを「道具セット」として登録
- 作業前と作業後にセット内容を確認
- 忘れ物を防止

**対策2: 移動時の記録を徹底**

- 現場から持ち出す際は必ずQRスキャン
- 返却時も必ずQRスキャン
- 「誰が・いつ・どこに」を記録

**対策3: 定期的な棚卸し**

- 月1回、倉庫の道具をすべて確認
- システム上の在庫数と実物を照合
- 不一致があれば即座に調査

**関連**: [棚卸し作業マニュアル](../../scenarios/admin_operations/inventory_audit.md)（作成予定）

---

### 5-2. 破損防止

**対策1: 適切な保管**

- 工具箱に収納（床に放置しない）
- 精密機器は専用ケースに保管
- 重機のアームに吊るさない

**対策2: 定期メンテナンス**

- 月1回、道具の点検を実施
- 異音・がたつきがあれば早めに修理
- 刃物は定期的に研磨

**関連**: [道具のメンテナンス・点検マニュアル](./tool_maintenance.md)

**対策3: 使用方法の教育**

- 新人に正しい使い方を教育
- 「こういう使い方はNG」を明示
- メーカーの取扱説明書を共有

## 6. よくある質問（FAQ）

### Q1. 道具が見つからないとき、すぐに「紛失」として処理すべきですか？

**A.** いいえ、まずは十分に捜索してください。

**推奨フロー**:
1. 30分〜1時間、自分で探す
2. 上司に報告し、組織的に捜索
3. 1〜3日捜索しても見つからない場合、紛失と判断
4. 管理者に削除を依頼

焦って削除すると、後で見つかった場合に混乱します。

---

### Q2. 破損した道具を修理に出す間、システム上でどう管理すればいいですか？

**A.** **「🔧 修理」** ステータスに変更してください。

**手順**:
1. 道具詳細画面の **「移動登録」** をクリック
2. **移動先**: **「🔧 修理」** を選択
3. **備考**: 修理業者名・修理内容・返却予定日を記入
4. **「登録する」** をクリック

これにより、道具一覧画面で **「現在位置: 修理中」** と表示され、他のスタッフにも分かります。

---

### Q3. 修理費が高額です。新品を買った方が良いでしょうか？

**A.** 一般的な判断基準は以下の通りです。

```
修理費 < 新品価格の50% → 修理推奨
修理費 ≥ 新品価格の50% → 新品購入を検討
修理費 > 新品価格 → 新品購入
```

ただし、以下の要素も考慮してください：
- 愛着のある道具か（長年使っている、使い慣れている）
- 修理後の耐久性（すぐまた壊れる可能性）
- 納期（修理1ヶ月 vs 新品は翌日届く）

**最終判断は上司と相談**してください。

---

### Q4. 道具を誤って削除してしまいました。復旧できますか？

**A.** 論理削除のため、システム管理者（Super Admin）であればデータベースから復旧可能です。

**対応**:
1. 即座に管理者に報告
2. 削除した道具のシリアル番号・削除日時を伝える
3. システム管理者がデータベースから `deleted_at` フィールドをNULLに戻す

ただし、**誤削除を防ぐため、削除操作は慎重に行ってください**。

---

### Q5. 現場で道具が盗まれました。警察に届けるべきですか？

**A.** はい、盗難の可能性がある場合は **必ず警察に届け出てください**。

**手順**:
1. **即座に上司・管理者に報告**
2. **警察に盗難届を提出**（受理番号を控える）
3. 近隣の工事現場・リサイクルショップに情報提供
4. 再発防止策を検討（保管場所の施錠、防犯カメラ設置など）

盗難届を出すことで、以下のメリットがあります：
- 警察が捜索してくれる
- 保険請求の証拠になる
- 再発防止の記録になる

## 7. チェックリスト

### 紛失時のチェックリスト

- [ ] 最後に使った場所を徹底的に探した
- [ ] システムで最終移動履歴を確認した
- [ ] チームメンバーに確認した
- [ ] 車両・トラックを確認した
- [ ] 上司に報告した
- [ ] 組織的な捜索を実施した（1〜3日）
- [ ] 警察に遺失物届を提出した（必要に応じて）
- [ ] 管理者に削除を依頼した（紛失確定後）
- [ ] 代替品を手配した

---

### 破損時のチェックリスト

- [ ] 使用を即座に中止した
- [ ] 安全上の危険がないか確認した
- [ ] 破損箇所の写真を撮影した
- [ ] 上司に報告した
- [ ] 修理見積を取得した
- [ ] 修理 or 新品購入の判断をした
- [ ] システム上で「修理中」に変更した（修理の場合）
- [ ] 修理業者に発送した
- [ ] 修理完了後、動作確認をした
- [ ] システム上で「倉庫」に戻した（修理完了後）

# 管理者向けクイックガイド

**最終更新日**: 2026-01-22
**対象**: admin（システム管理者）役割のユーザー

---

## はじめに

このガイドは、システム管理者（admin）の方がザイロクを使い始めるための基本操作をまとめたクイックガイドです。

### 管理者ができること

管理者は、組織全体を管理する最高権限者です。マネージャーができることに加えて、以下の操作ができます：

✅ **マネージャーの機能に加えて**:
- 道具・消耗品・重機の削除
- 倉庫位置の管理
- 取引先マスタの管理
- マスタ管理（カテゴリ・道具・消耗品）の削除
- すべてのスタッフの追加・編集・削除
- 権限変更（役割の変更）
- パスワードリセット
- 出退勤設定
- 作業報告書設定
- 組織情報設定
- 運用設定
- 監査ログの閲覧

❌ **できないこと**:
- 他組織のデータへのアクセス

## 管理者の責任

管理者は、組織全体のザイロク運用を管理する重要な役割です。

### 主な責任

1. **初期設定**
   - 組織情報の設定
   - 運用設定の設定
   - 倉庫位置の登録
   - カテゴリの設定

2. **ユーザー管理**
   - スタッフの追加・編集・削除
   - 権限の変更
   - パスワードリセット

3. **マスタ管理**
   - 取引先の管理
   - 道具カテゴリの管理
   - 倉庫位置の管理

4. **運用管理**
   - 出退勤設定
   - 作業報告書設定
   - QR更新頻度の設定

5. **監視**
   - 監査ログの確認
   - 不正アクセスの検知
   - セキュリティインシデントへの対応

### 推奨: 複数の管理者を設定

組織には2-3人の管理者を設定することを推奨します。

**理由**:
- 管理者が休暇や退職した場合のリスク回避
- 緊急時の対応
- 相互監視によるセキュリティ強化

## 初期設定

### 1. 組織情報を設定する

1. サイドメニュー → **「運用設定」** → **「組織情報」**
2. 以下の項目を設定
   - **QRスキャン設定**
     - 道具移動時にQRスキャン必須
     - 道具返却時のみQRスキャン必須
   - **紛失・承認設定**
     - 紛失承認フロー
   - **通知・リマインド設定**
     - 低在庫アラート通知
     - 月次棚卸しリマインド通知
     - 現場終了時のチェックリスト
   - **QRコード設定**
     - QRコード印刷サイズ（2.5cm × 2.5cm 推奨）
   - **消耗品管理設定**
     - 消耗品の移動記録方式（パターンA推奨）
   - **重機機能設定**
     - 重機管理機能を有効にする（必要に応じて）
   - **倉庫階層設定**
     - 倉庫階層を設定（3階層推奨）
3. **「設定を保存」** をクリック

**詳細**: [組織情報設定Q&A](../../../qa/admin/settings_organization_issues.md)を参照

### 2. 倉庫位置を登録する

1. サイドメニュー → **「倉庫・現場」** → **「倉庫位置」**
2. **「新規登録」** ボタンをクリック
3. 必要事項を入力
   - 名前（例: 倉庫A-1棚）
   - 階層情報（設定した階層に応じて）
4. **「登録」** をクリック

**推奨**: 倉庫の配置図を用意して、位置コードを決めてから登録

### 3. 道具カテゴリを設定する

**対象**: 現場資産パック・フル機能統合パックをご契約の組織のみ

1. サイドメニュー → **「マスタ管理」** → **「道具カテゴリ」**
2. **「新規登録」** ボタンをクリック
3. カテゴリ名を入力（例: 電動工具、手動工具、測定機器）
4. **「登録」** をクリック

## スタッフを管理する

### スタッフを追加する

1. サイドメニュー → **「設定」** → **「スタッフ管理」**
2. **「スタッフを追加」** ボタンをクリック
3. 必要事項を入力
   - 氏名
   - メールアドレス
   - 役割（staff/leader/manager/admin）
   - 所属部署（オプション）
4. **「招待メールを送信」** をクリック

**注意**:
- 管理者は、すべての役割のスタッフを追加できます
- 招待メールが送信され、スタッフが初回パスワードを設定します

### 権限を変更する

1. スタッフ一覧から対象スタッフをクリック
2. **「編集」** ボタンをクリック
3. **「役割」** を変更
4. **「保存」** をクリック

**注意**:
- 権限変更は慎重に行ってください
- admin権限を与えると、システム設定が変更できるようになります

### パスワードをリセットする

1. スタッフ一覧から対象スタッフをクリック
2. **「パスワードリセット」** ボタンをクリック
3. 確認ダイアログで **「はい」** をクリック
4. パスワードリセットリンクがスタッフのメールアドレスに送信される

### スタッフを削除（無効化）する

1. スタッフ一覧から対象スタッフをクリック
2. **「無効化」** ボタンをクリック
3. 確認ダイアログで **「はい」** をクリック

**注意**:
- 無効化されたスタッフはログインできなくなります
- データは削除されず、操作履歴は保持されます
- 再度有効化することも可能です

**詳細**: [スタッフ管理マニュアル](../../staff_management.md)を参照

## 組織情報を設定する

### QRスキャン設定

**道具移動時にQRスキャン必須**:
- オン: 道具の持ち出し・返却時に必ずQRスキャンが必要
- オフ: 手動で道具を選択することも可能

**推奨**: 最初はオフで運用に慣れてから、オンに変更

**道具返却時のみQRスキャン必須**:
- オン: 返却時のみQRスキャンが必要
- オフ: 返却時も手動選択可能

**推奨**: オン（返却管理を重視）

### 紛失承認フロー

**紛失承認フローを有効にする**:
- オン: 紛失報告は管理者の承認が必要
- オフ: 紛失報告は即座に確定

**推奨**: オン（誤報を防ぐ）

### QRコード印刷サイズ

**選択肢**:
- 1.2cm × 1.2cm（極小）
- 2cm × 2cm（小）
- 2.5cm × 2.5cm（推奨）
- 3cm × 3cm（中）
- 5cm × 5cm（大）

**推奨**: 2.5cm × 2.5cm（読み取りやすさとサイズのバランスが良い）

### 倉庫階層設定

**推奨設定**: 3階層
- レベル1: エリア（A、B、C など）
- レベル2: 棚（1、2、3 など）
- レベル3: 段（上、中、下 など）

**位置コード例**: A-1-上

**詳細**: [組織情報設定Q&A](../../../qa/admin/settings_organization_issues.md)を参照

## 倉庫位置を管理する

### 倉庫位置を登録する

1. サイドメニュー → **「倉庫・現場」** → **「倉庫位置」**
2. **「新規登録」** ボタンをクリック
3. 必要事項を入力
   - 名前（例: 倉庫A-1棚）
   - 階層情報
   - 説明（オプション）
4. **「登録」** をクリック

### 倉庫位置を編集する

1. 倉庫位置一覧から対象位置をクリック
2. **「編集」** ボタンをクリック
3. 必要な項目を修正
4. **「保存」** をクリック

### 倉庫位置を削除する

1. 倉庫位置一覧から対象位置をクリック
2. **「削除」** ボタンをクリック
3. 確認ダイアログで **「はい」** をクリック

**注意**:
- 道具が保管されている倉庫位置は削除できません
- 先に道具を移動してから削除してください

**詳細**: [倉庫位置管理マニュアル](../../warehouse_locations.md)を参照

## 出退勤設定を行う

**対象**: DX効率化パック・フル機能統合パックをご契約の組織のみ

### 出退勤設定画面

1. サイドメニュー → **「出退勤管理」** → **「出退勤設定」**
2. 以下の項目を設定

### QRスキャン設定

**QRスキャン許可**:
- オン: QRスキャンで出退勤打刻が可能
- オフ: 手動打刻のみ

**QR更新頻度**:
- 1日/3日/7日/30日（推奨）

**推奨**: 30日（毎月1回の再発行で管理が簡単）

### 打刻設定

**打刻時刻の丸め**:
- なし/15分単位/30分単位

**休憩時間の自動控除**:
- オン: 指定時間の休憩を自動控除
- オフ: 控除なし

### アラート設定

**打刻忘れアラート**:
- オン: 打刻忘れを通知
- オフ: 通知なし

**遅刻アラート**:
- オン: 遅刻を通知
- オフ: 通知なし

3. **「設定を保存」** をクリック

**詳細**: [出退勤設定Q&A](../../../qa/admin/settings_attendance_issues.md)を参照

## 監査ログを確認する

### 監査ログとは

システム内のすべての操作記録を自動的に記録する機能です。

**記録される内容**:
- ログイン履歴（成功・失敗）
- IPアドレス
- ユーザーエージェント（ブラウザ情報）
- 操作内容（道具の登録、移動、編集、削除など）
- 操作日時

### 監査ログの確認方法

1. サイドメニュー → **「設定」** → **「監査ログ」**
2. 監査ログ一覧が表示される

**フィルター機能**:
- 期間で絞り込み
- ユーザーで絞り込み
- 操作種類で絞り込み

### 不正アクセスの検出

**確認すべき項目**:
- ログイン失敗の多発
- 通常と異なるIPアドレスからのアクセス
- 深夜の操作
- 不審な削除操作

**対応**:
- 該当アカウントを無効化
- パスワードをリセット
- IP制限を有効化

**詳細**: [緊急対応マニュアル](../security/emergency_response.md)を参照

## よくある質問

### Q1. 他の管理者を追加したい

**A.** スタッフ管理から追加してください：
1. スタッフ管理 → スタッフを追加
2. 役割で **「admin」** を選択
3. 招待メールを送信

**推奨**: 2-3人の管理者を設定

### Q2. スタッフのパスワードを知りたい

**A.** セキュリティ上、パスワードは管理者も知ることができません。
- パスワードを忘れた場合は、パスワードリセットを実行してください

### Q3. 削除した道具を復元したい

**A.** 削除した道具は復元できません。
- 削除前に、本当に削除してよいか確認してください
- 必要に応じて、バックアップからの復元を検討してください

### Q4. 組織名を変更したい

**A.** 組織情報設定画面では変更できません。
- サポートに連絡して変更を依頼してください

### Q5. IP制限を有効にしたら、自分もログインできなくなりました

**A.** 以下の対処方法があります：
- 許可されたネットワークに接続
- 他の管理者に依頼して、自分のIPアドレスを追加してもらう
- システム管理者（サービス提供者）に連絡

**詳細**: [IP制限Q&A](../../../qa/admin/settings_organization_issues.md#q46-ip制限を有効にしたら自分もログインできなくなりました)を参照

### Q6. セキュリティインシデントが発生しました

**A.** 緊急対応手順を実施してください：
1. 該当アカウントを無効化（5分以内）
2. 他の管理者に報告
3. 監査ログで不正操作を確認
4. 必要に応じて、組織全体のパスワードリセット

**詳細**: [緊急対応マニュアル](../security/emergency_response.md)を参照

### Q7. 退職者のアカウントはどうすればよいですか？

**A.** 無効化してください：
1. スタッフ管理 → 対象スタッフ
2. **「無効化」** ボタンをクリック

**注意**: 削除ではなく、無効化を推奨（操作履歴を保持）

**詳細**: [退職者アカウントの処理](../../../qa/admin/settings_organization_issues.md#q55-退職者のアカウントはどう処理すればよいですか)を参照

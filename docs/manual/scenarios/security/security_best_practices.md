# セキュリティのベストプラクティス

## 1. 概要

ザイロクを安全に運用するためのセキュリティのベストプラクティスを説明します。パスワード管理、アクセス制御、監査ログの活用など、組織のデータを守るための推奨事項を網羅しています。

**このマニュアルで解決できること**:
- 強固なパスワードポリシーの設定
- 2段階認証の導入と管理
- 適切な権限設定
- ログイン制限とIP制限の活用
- 監査ログによる操作追跡

**対象者**: Admin（管理者）、すべてのユーザー

---

## 2. パスワードセキュリティ

### 2-1. 強固なパスワードの作成

**推奨パスワードポリシー**:

| 項目 | 推奨値 | 理由 |
|------|--------|------|
| **最小文字数** | 12文字以上 | 総当たり攻撃に強い |
| **文字種** | 大文字・小文字・数字・記号を含む | 推測されにくい |
| **辞書単語の禁止** | パスワードに辞書の単語を使わない | 辞書攻撃に強い |
| **個人情報の禁止** | 名前、誕生日、電話番号を使わない | 推測されにくい |
| **使い回しの禁止** | 他のサービスと同じパスワードを使わない | 漏洩時の被害を最小化 |

**良いパスワードの例**:
```
✅ Good: K9$mPq2@xL7nR4
✅ Good: Tr0ub!e#Maker2024
✅ Good: Ze@ro-D@y!Sec9

❌ Bad: password123
❌ Bad: yamada2024
❌ Bad: 12345678
```

---

### 2-2. パスワード管理のベストプラクティス

#### ✅ すべきこと

1. **定期的なパスワード変更**
   - 推奨: 3ヶ月〜6ヶ月ごと
   - Admin権限: 3ヶ月ごと
   - 一般スタッフ: 6ヶ月ごと

2. **パスワードマネージャーの使用**
   - 1Password、LastPass、Bitwardenなどの利用を推奨
   - 複雑なパスワードを安全に保管

3. **初期パスワードの即時変更**
   - 新入社員は初回ログイン後すぐにパスワード変更
   - 初期パスワードは一時的なものとして扱う

4. **パスワードの安全な保管**
   - 紙に書いて鍵付きキャビネットに保管（物理的なバックアップ）
   - メールやチャットで送信しない

#### ❌ してはいけないこと

1. **パスワードの共有**
   - 同僚とパスワードを共有しない
   - 家族にも教えない

2. **パスワードの使い回し**
   - 他のサービス（Gmail、Facebookなど）と同じパスワードを使わない

3. **簡単なパスワード**
   - 「password」「123456」などの単純なパスワード
   - 会社名や個人名のみのパスワード

4. **公共の場でのパスワード入力**
   - カフェなどの公共Wi-Fiでのログインは避ける
   - やむを得ない場合はVPNを使用

## 3. 2段階認証（2FA）

### 3-1. 2FAとは

2段階認証（Two-Factor Authentication）は、パスワードに加えて「もう1つの認証要素」を要求することで、アカウントのセキュリティを大幅に向上させます。

**認証要素の種類**:
1. **知識要素**: パスワード（あなたが知っているもの）
2. **所持要素**: スマートフォン、認証アプリ（あなたが持っているもの）

---

### 3-2. 2FAの種類

ザイロクでは2種類の2FA方式をサポートしています。

#### 方式1: TOTP（Time-based One-Time Password）

**仕組み**:
- Google Authenticator、Microsoft Authenticator、Authyなどの認証アプリを使用
- 30秒ごとに変わる6桁のコードを生成
- オフラインでも使用可能

**設定方法**:
1. 右上のアイコン → **「アカウント設定」**
2. **「セキュリティ」** タブをクリック
3. **「2段階認証を有効化」** をクリック
4. 認証方式で **「認証アプリ」** を選択
5. QRコードをGoogle Authenticatorでスキャン
6. 6桁のコードを入力して確認

**推奨**: Admin、Manager権限には必須

---

#### 方式2: Email（メール認証）

**仕組み**:
- ログイン時に登録メールアドレスに6桁のコードを送信
- コードの有効期限は10分間

**設定方法**:
1. 右上のアイコン → **「アカウント設定」**
2. **「セキュリティ」** タブをクリック
3. **「2段階認証を有効化」** をクリック
4. 認証方式で **「メール」** を選択
5. 認証用メールアドレスを入力（会社のメールアドレス推奨）
6. テストコードを受信して確認

**注意**: メールアカウントが乗っ取られると2FAも突破されるため、メールアカウントも強固なパスワードで保護してください。

---

### 3-3. 2FA必須ポリシーの設定

Admin権限者は、組織全体で2FAを必須にできます。

**設定方法**:
1. 右上のアイコン → **「管理者メニュー」** → **「システム設定」**
2. **「セキュリティ設定」** セクションを開く
3. **「2段階認証の必須化」** で以下を設定:
   - 組織管理者（Admin）: ✅ 推奨
   - マネージャー（Manager）: ✅ 推奨
   - リーダー（Leader）: 任意
   - スタッフ（Staff）: 任意
4. **「保存」** をクリック

**効果**: 設定した権限のユーザーは、次回ログイン時に2FAの設定が必須になります（未設定の場合は警告が表示されます）。

## 4. アクセス制御

### 4-1. 権限の最小化原則

**原則**: ユーザーには業務に必要な最小限の権限のみを付与する。

| 権限 | 付与対象 | できること |
|------|---------|-----------|
| **Staff** | 現場作業員、技能職 | 道具の貸出・返却、作業報告書作成 |
| **Leader** | 班長、職長 | Staffの権限 + 道具セット作成、チーム管理 |
| **Manager** | 現場監督、部門責任者 | Leaderの権限 + 見積書・請求書承認 |
| **Admin** | 経営陣、システム管理者 | 全権限 + スタッフ管理、組織設定 |

**ベストプラクティス**:
- 新入社員は**Staff**からスタート
- 昇進・役職変更時のみ権限を昇格
- 退職者・休職者の権限は即座に無効化
- Admin権限は2〜3名に限定

---

### 4-2. Admin権限の管理

**Admin権限は強力です**。以下に注意してください：

#### 注意事項

1. **Admin権限者の数を最小限に**
   - 推奨: 2〜3名
   - 多すぎるとセキュリティリスクが増加

2. **Admin権限者の選定基準**
   - 信頼できる人物
   - ITリテラシーが高い
   - 長期勤続者

3. **Admin権限者の退職時**
   - 退職日当日に無効化
   - 退職前に後任のAdmin権限者を任命
   - Admin権限者が0名になると設定変更ができなくなる

---

### 4-3. 定期的な権限レビュー

**推奨頻度**: 3ヶ月ごと

**確認項目**:
- [ ] 退職者・休職者のアカウントが無効化されているか
- [ ] 不要な権限を持つスタッフがいないか
- [ ] Admin権限者が適切か
- [ ] 長期間ログインしていないアカウントがないか

**確認方法**:
1. 左サイドメニュー **「スタッフ」** をクリック
2. **「フィルタ」** で権限別に確認
3. 不要なアカウントは無効化

**詳細**: [退職者の対応マニュアル](../admin_operations/employee_offboarding.md)

## 5. ログイン制限とIP制限

### 5-1. ログイン試行回数制限

ザイロクは自動的にログイン試行回数を制限します。

**デフォルト設定**:
- **最大試行回数**: 5回
- **ロックアウト期間**: 30分間
- **警告表示**: 残り3回以下で表示

**仕組み**:
1. 同じメールアドレスで5回ログイン失敗
2. アカウントが30分間ロック
3. ロック中はログインできない（正しいパスワードでも）
4. 30分後に自動解除

**対応**:
- ログイン試行回数制限は変更できません（セキュリティのため）
- ロック解除を急ぐ場合は、Admin権限者に連絡してパスワードリセットを依頼

---

### 5-2. IP制限（ホワイトリスト方式）

特定のIPアドレスからのみアクセスを許可できます。

**使用例**:
- オフィスのIPアドレスのみ許可
- VPNのIPアドレスのみ許可
- 特定の顧客ネットワークのみ許可

**設定方法**:
1. 右上のアイコン → **「管理者メニュー」** → **「システム設定」**
2. **「セキュリティ設定」** セクションを開く
3. **「IPアドレス制限を有効化」** をON
4. **「許可するIPアドレス」** に追加:
   - 完全一致: `203.0.113.1`
   - ワイルドカード: `203.0.113.*`
   - CIDR記法: `203.0.113.0/24`
5. **「保存」** をクリック

**注意事項**:
- ⚠️ 設定ミスで自分自身もロックアウトされる可能性があります
- ⚠️ テスト環境で十分に確認してから本番環境で設定してください
- ⚠️ 動的IPアドレス（家庭用インターネット）は使用しないでください

**推奨**: 固定IPアドレスを持つオフィスやVPN環境でのみ使用

## 6. 監査ログの活用

### 6-1. 監査ログとは

監査ログは、ザイロク上のすべての重要な操作を記録します。

**記録される情報**:
- ログイン・ログアウト
- スタッフの作成・編集・削除
- 道具の作成・編集・削除
- 権限の変更
- システム設定の変更
- IPアドレス、User-Agent

---

### 6-2. 監査ログの確認方法

**アクセス方法**:
1. 右上のアイコン → **「管理者メニュー」** → **「監査ログ」**

**フィルタ機能**:
- **アクション**: 作成、更新、削除
- **エンティティ種別**: スタッフ、道具、現場など
- **ユーザー**: 特定のスタッフの操作のみ表示
- **期間**: 開始日〜終了日で絞り込み

**使用例**:

**例1: 特定のスタッフの操作履歴を確認**
```
1. ユーザーフィルタで「山田太郎」を選択
2. 期間を「1週間前〜今日」に設定
3. 山田太郎の全操作が表示される
```

**例2: 削除操作のみを確認**
```
1. アクションフィルタで「削除」を選択
2. 誰がいつ何を削除したかが一覧表示
```

**例3: 不審なログインを調査**
```
1. アクションフィルタで「ログイン」を選択
2. 深夜（22:00〜6:00）のログインを手動で確認
3. 本人に確認
```

---

### 6-3. 監査ログの定期確認

**推奨頻度**: 月1回

**確認項目**:
- [ ] 深夜・早朝の不審なログイン
- [ ] 退職者のログイン（無効化漏れ）
- [ ] 大量の削除操作
- [ ] 権限の不正変更
- [ ] 同じIPアドレスから複数アカウントのログイン（アカウント共有の可能性）

**詳細**: [システム健全性チェックマニュアル](../admin_operations/system_health_check.md)

## 7. セキュアな運用習慣

### 7-1. ログイン・ログアウトの習慣

#### ✅ すべきこと

1. **共用PCでは必ずログアウト**
   - 作業終了後は右上のアイコン → **「ログアウト」**
   - ブラウザを閉じるだけではログアウトされません

2. **長時間離席時はログアウトまたは画面ロック**
   - 1時間以上離席する場合はログアウト
   - 短時間の場合はPCの画面ロック（Windows: Win+L、Mac: Control+Command+Q）

3. **スマートフォンのロック**
   - スマホでザイロクにアクセスする場合、必ず画面ロックを設定
   - 紛失・盗難時のリスクを軽減

---

### 7-2. フィッシング詐欺対策

**フィッシング詐欺とは**:
偽のログイン画面に誘導し、パスワードを盗む攻撃

**見分け方**:
- ✅ 正規URL: `https://zairoku.com/login`
- ❌ 偽URL: `https://zairoku-login.com` (ドメインが違う)
- ❌ 偽URL: `https://zairoku.com.evil.com` (サブドメイン)

**対策**:
1. **URLを必ず確認**
   - ブラウザのアドレスバーに `zairoku.com` と表示されているか確認
2. **ブックマークからアクセス**
   - メールのリンクからアクセスしない
3. **HTTPSを確認**
   - アドレスバーに鍵マークがあることを確認

**怪しいメールの特徴**:
- 「パスワードが期限切れです。すぐに更新してください」
- 「不正なログインが検出されました。確認してください」
- 差出人が `@zairoku.com` ではない

**ザイロクからの正規メール**:
- 差出人: `noreply@zairoku.com`
- 内容: 2FA認証コードのみ（パスワード入力を求めることはありません）

---

### 7-3. 公共Wi-Fiでの使用

**リスク**:
- 通信が盗聴される可能性
- 中間者攻撃（Man-in-the-Middle）

**対策**:
1. **可能な限り避ける**
   - カフェ、空港、ホテルなどの公共Wi-Fiでは重要な操作をしない
2. **VPNを使用**
   - 会社のVPNまたは商用VPN（NordVPN、ExpressVPNなど）を使用
3. **モバイルデータを使用**
   - スマートフォンのテザリングを使用

---

## 8. データ保護

### 8-1. データの閲覧制限

**原則**: 業務に必要な情報のみ閲覧する

**例**:
- 自分が担当する現場の情報のみ閲覧
- 他のスタッフの個人情報は必要な場合のみ閲覧
- 見積書・請求書は担当者・承認者のみ閲覧

---

### 8-2. スクリーンショットと印刷

**注意事項**:
- スクリーンショットは業務上必要な場合のみ
- 印刷物は机に放置しない
- 不要になった印刷物はシュレッダーで廃棄
- スクリーンショットをSNSに投稿しない

---

## 9. インシデント対応

### 9-1. セキュリティインシデントの種類

| インシデント | 例 | 対応 |
|------------|-----|------|
| **不正ログイン** | 自分がログインしていない時間帯のログイン記録 | Admin権限者に即座に報告 |
| **パスワード漏洩** | パスワードを第三者に知られた | 即座にパスワード変更 |
| **アカウント乗っ取り** | 勝手にパスワードが変更された | Admin権限者に連絡してアカウント無効化 |
| **データ漏洩** | 顧客情報を誤って外部に送信 | 上司・Admin権限者に報告 |

**詳細**: [緊急時の対応マニュアル](./emergency_response.md)

---

### 9-2. 報告フロー

**インシデント発生時**:
1. **即座に上司またはAdmin権限者に報告**
2. **被害を最小化**（アカウント無効化、パスワード変更など）
3. **記録を残す**（何が起きたか、いつ起きたか）
4. **再発防止策を検討**

**報告先**:
- 小規模インシデント: 上司
- 重大インシデント: Admin権限者 + 経営層

## 10. チェックリスト

### 個人ユーザー向けセキュリティチェックリスト

- [ ] 強固なパスワードを使用している（12文字以上、複雑）
- [ ] パスワードを他のサービスと使い回していない
- [ ] 2段階認証を設定している
- [ ] パスワードを3〜6ヶ月ごとに変更している
- [ ] 共用PCでは必ずログアウトしている
- [ ] フィッシング詐欺に注意している
- [ ] 公共Wi-Fiでの重要な操作を避けている
- [ ] スマートフォンに画面ロックを設定している

---

### Admin権限者向けセキュリティチェックリスト

- [ ] 組織全体の2FA必須ポリシーを設定している（Admin、Manager）
- [ ] ログイン試行回数制限が有効になっている（デフォルトで有効）
- [ ] IP制限を検討している（固定IPがある場合）
- [ ] 監査ログを月1回確認している
- [ ] 退職者のアカウントを即座に無効化している
- [ ] Admin権限者が2〜3名に限定されている
- [ ] 権限レビューを3ヶ月ごとに実施している
- [ ] スタッフにセキュリティ教育を実施している

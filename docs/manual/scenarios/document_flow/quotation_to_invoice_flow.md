# 見積→契約→請求の一連の流れ

## 概要

このマニュアルでは、顧客から依頼を受けて見積書を作成し、契約が成立した後に請求書を発行し、入金を記録するまでの一連の業務フローを説明します。

## 対象者

- リーダー以上: 見積書・請求書の作成・提出
- マネージャー以上: 承認操作
- 管理者: すべての操作

## 業務フロー全体図

```
顧客からの依頼
    ↓
【STEP 1】見積書作成
    ↓
【STEP 2】見積書提出・承認
    ↓
【STEP 3】見積書送付（顧客へ）
    ↓
顧客からの承諾（契約成立）
    ↓
【STEP 4】契約書登録（任意）
    ↓
【STEP 5】請求書作成（見積書から変換）
    ↓
【STEP 6】請求書提出・承認
    ↓
【STEP 7】請求書送付（顧客へ）
    ↓
【STEP 8】入金記録
    ↓
完了
```

---

## STEP 1: 見積書作成

### 目的
顧客に提示する工事費用・資材費用の見積書を作成します。

### 操作手順

#### 1. 見積書作成画面を開く

1. サイドメニューから「見積書」をクリック
2. 「+ 新規見積書」ボタンをクリック

#### 2. 基本情報を入力

| 項目 | 説明 | 入力例 |
|------|------|--------|
| **見積番号** | 自動生成（編集不可） | EST-202601-0001 |
| **取引先** | 顧客を選択（必須） | A建設株式会社 |
| **件名** | 見積内容の概要（必須） | 〇〇ビル外壁塗装工事 |
| **見積日** | 見積書の発行日（必須） | 2026-01-15 |
| **有効期限** | 見積の有効期限（必須） | 2026-02-14（30日後） |
| **現場** | 紐付ける現場（任意） | 〇〇ビル現場 |

#### 3. 見積明細を入力

「+ 明細を追加」ボタンをクリックして、品目ごとに入力します。

**入力例**：
```
品目: 外壁塗装（1階部分）
数量: 1
単位: 式
単価: 500,000
金額: 500,000（自動計算）

品目: 外壁塗装（2階部分）
数量: 1
単位: 式
単価: 450,000
金額: 450,000（自動計算）

品目: 足場組立・解体
数量: 1
単位: 式
単価: 200,000
金額: 200,000（自動計算）
```

#### 4. 合計金額を確認

- 小計: ¥1,150,000
- 消費税（10%）: ¥115,000
- **合計金額: ¥1,265,000**

#### 5. 備考を入力（任意）

```
・作業期間: 2026年3月1日〜3月31日（予定）
・雨天の場合は作業を延期する場合があります
・足場設置期間中は駐車場の一部が使用できません
```

#### 6. 保存

- 「下書き保存」: 提出せずに保存（後で編集可能）
- 「提出」: 承認待ちステータスに移行

---

## STEP 2: 見積書提出・承認

### 目的
作成した見積書を上長に承認してもらいます。

### 操作手順

#### 作成者（リーダー）の操作

1. 見積書詳細画面で「提出」ボタンをクリック
2. ステータスが「提出済み」に変更
3. 承認者（マネージャー・管理者）に通知が送信される

#### 承認者（マネージャー・管理者）の操作

1. 通知またはダッシュボードから「承認待ち見積書」を確認
2. 見積書詳細画面を開く
3. 内容を確認
4. **承認する場合**: 「承認」ボタンをクリック
5. **却下する場合**: 「却下」ボタンをクリック → 却下理由を入力

### 承認後のステータス

- 承認: ステータスが「承認済み」に変更
- 却下: ステータスが「却下」に変更 → 作成者が修正して再提出

---

## STEP 3: 見積書送付（顧客へ）

### 目的
承認済みの見積書を顧客に送付します。

### 操作手順

#### 1. PDF出力

1. 見積書詳細画面を開く
2. 「PDFダウンロード」ボタンをクリック
3. PDFファイルがダウンロードされる

**PDFに含まれる内容**：
- 会社印（角印）※組織設定で登録している場合のみ
- 見積明細
- 合計金額
- 備考

#### 2. 顧客へ送付

**方法1: メール送付**
- PDFファイルを添付してメールで送付

**方法2: 郵送**
- PDFを印刷して郵送

**方法3: 送付済みステータスに変更**
1. 見積書詳細画面で「送付済みにする」ボタンをクリック
2. 確認ダイアログで「OK」をクリック
3. ステータスが「顧客送付済み」に変更
4. 送付日が記録される

**注意**: このボタンを押してもメール送信等は行われません。実際に顧客へ送付した後に、記録のために使用してください。

---

## STEP 4: 契約書登録（任意）

### 目的
顧客から承諾を得て契約が成立したことを記録します。

### 操作手順

#### 契約の記録方法

契約が成立したら、見積書のステータスを「顧客承認」に変更します。

1. 見積書詳細画面を開く
2. 「顧客承認」ボタンをクリック
3. ステータスが「顧客承認」に変更される

**注意事項**：
- 「顧客承認」ボタンはマネージャーまたは管理者のみが使用できます
- 見積書のステータスが「顧客送付済み」の状態で使用できます
- この操作により、見積書から請求書への変換が可能になります

**契約書ファイルの管理（オプション）**：
- 紙の契約書や電子契約のPDFは、見積書の備考欄にファイル名や保管場所を記載してください
- または、社内の文書管理システムで別途管理してください

---

## STEP 5: 請求書作成（見積書から変換）

### 目的
契約が成立した見積書から請求書を作成します。

### 前提条件
- 見積書のステータスが「顧客承認」になっていること

### 操作手順

#### 1. 見積書から請求書に変換

1. 見積書詳細画面を開く
2. 「請求書作成」ボタンをクリック（「顧客承認」ステータスの場合のみ表示）
3. 請求書作成画面が開く
4. 見積書のデータが自動入力される

**自動入力される項目**：
- 取引先
- 件名
- 明細（品目、数量、単価、金額）
- 備考

#### 2. 請求書固有の情報を入力

| 項目 | 説明 | 入力例 |
|------|------|--------|
| **請求番号** | 自動生成 | INV-202603-0001 |
| **請求日** | 請求書の発行日 | 2026-03-31 |
| **支払期日** | 入金期限 | 2026-04-30（30日後） |
| **現場** | 紐付ける現場 | 〇〇ビル現場 |

#### 3. 明細を確認・修正

見積書から自動入力された明細を確認します。

**修正が必要な場合**：
- 工事内容が変更された場合
- 追加工事が発生した場合
- 値引きを適用する場合

#### 4. 保存

- 「下書き保存」: 提出せずに保存
- 「提出」: 承認待ちステータスに移行

---

3. 承認者に通知が送信される

#### 承認者（マネージャー・管理者）の操作

### 承認後のステータス

- 却下: ステータスが「却下」に変更 → 作成者が修正して再提出

## STEP 8: 入金記録

### 目的
顧客からの入金を記録します。

### 操作手順

#### 1. 入金記録画面を開く

**方法1: 請求書詳細画面から**
1. 請求書詳細画面を開く
2. 「入金登録」ボタンをクリック

**方法2: 入金一覧から**
1. サイドメニューから「入金・支払」をクリック
2. 「+ 新規登録」ボタンをクリック
3. 支払タイプで「入金」を選択

#### 2. 入金情報を入力

| 項目 | 説明 | 入力例 |
|------|------|--------|
| **入金日** | 実際に入金があった日 | 2026-04-28 |
| **請求書** | 入金対象の請求書を選択 | INV-202603-0001 |
| **入金額** | 今回入金された金額 | 1,265,000 |
| **支払方法** | 銀行振込、現金など | 銀行振込 |
| **参照番号** | 振込依頼人名など（任意） | A建設(株) |
| **メモ** | 補足情報（任意） | 支払期日通り入金 |

#### 3. 登録

「登録」ボタンをクリックすると、入金が記録されます。

### 入金後のステータス変化

- **全額入金の場合**: 請求書のステータスが「送付済み」→「支払済み」に変更
- **部分入金の場合**: ステータスは「送付済み」のまま（残額が表示される）

---

## 部分入金の場合

### シナリオ

請求金額: ¥1,265,000
1回目入金: ¥500,000
2回目入金: ¥765,000

### 操作手順

#### 1回目の入金記録

1. 入金日: 2026-04-15
2. 入金額: ¥500,000
3. 登録後:
   - 既入金額: ¥500,000
   - 残額: ¥765,000
   - ステータス: 送付済み（変化なし）

#### 2回目の入金記録

1. 入金日: 2026-04-30
2. 入金額: ¥765,000
3. 登録後:
   - 既入金額: ¥1,265,000
   - 残額: ¥0
   - ステータス: 支払済み（自動変更）

## よくあるケース

### ケース1: 見積金額と請求金額が異なる場合

**シナリオ**: 工事中に追加工事が発生した

**対処法**:
1. 見積書から請求書に変換
2. 請求書の明細に追加工事の行を追加
3. 合計金額を確認
4. 提出・承認フローに進む

**例**:
```
【見積書】
外壁塗装: ¥1,150,000
足場: ¥200,000
小計: ¥1,350,000
消費税: ¥135,000
合計: ¥1,485,000

【請求書】
外壁塗装: ¥1,150,000
足場: ¥200,000
追加工事（雨どい修理）: ¥100,000 ← 追加
小計: ¥1,450,000
消費税: ¥145,000
合計: ¥1,595,000
```

### ケース2: 値引きを適用する場合

**シナリオ**: 顧客に値引きを適用して請求

**対処法**:
1. 見積書から請求書に変換
2. 明細の最後に「値引き」行を追加
3. 数量: 1、単価: -50,000（マイナス値）
4. 合計金額を確認

**例**:
```
外壁塗装: ¥1,150,000
足場: ¥200,000
値引き: -¥50,000 ← マイナス値で入力
小計: ¥1,300,000
消費税: ¥130,000
合計: ¥1,430,000
```

### ケース3: 見積書なしで請求書を直接作成

**シナリオ**: 小規模な修繕工事で見積書を省略

**対処法**:
1. サイドメニューから「請求書」→「+ 新規請求書」
2. 基本情報と明細を直接入力
3. 提出・承認フローに進む

## チェックリスト

### 見積書作成時

- [ ] 取引先が正しく選択されているか
- [ ] 件名が具体的に記載されているか
- [ ] 見積日・有効期限が適切か
- [ ] 明細の品目・数量・単価が正確か
- [ ] 合計金額が妥当か
- [ ] 備考に必要事項が記載されているか

- [ ] 見積書の内容と一致しているか（または変更理由が明確か）

- [ ] 追加工事がある場合、明細に反映されているか
- [ ] 合計金額が正確か

### 入金記録時

- [ ] 入金日が正しいか（実際に入金された日）
- [ ] 入金額が正確か
- [ ] 支払方法が正しく選択されているか
- [ ] 参照番号（振込依頼人名など）が記録されているか

## トラブルシューティング

### Q: 見積書を承認してから内容を変更したい

**A**: 承認済みの見積書は編集できません。以下の対応を取ってください：
1. 見積書を複製して新しい見積書を作成
2. 内容を修正
3. 再度提出・承認フローに進む

### Q: 請求書の明細を間違えて登録してしまった

**A**: ステータスによって対応が異なります：
- **下書き中**: 編集可能です。請求書詳細画面から「編集」ボタンをクリック
- **提出済み・承認済み**: 編集できません。管理者に相談して、請求書を却下してもらい、再作成してください

### Q: 入金額を間違えて記録してしまった

**A**: 入金記録は編集できます。以下の手順で修正してください：
1. 入金・支払一覧画面を開く
2. 該当の入金記録をクリック
3. 「編集」ボタンをクリック
4. 正しい金額を入力して保存

**注意**: 編集はマネージャー以上の権限が必要です。

# 管理者・リーダー向け 操作マニュアル

> **対象ユーザー**: 管理者・現場リーダーの方
> **使用デバイス**: PC（推奨）、タブレット

---

## 🖥️ PCでの基本操作

### アプリへのアクセス
1. PCのブラウザ（Chrome、Edge、Safari等）を開く
2. 会社のシステムURL（例：`https://yourcompany.example.com`）にアクセス
3. 管理者アカウントでログイン

## 🔧 道具マスタの管理

### 新しい道具を登録する

1. サイドバーの「道具管理 ▼」→「🔧 個別管理道具」をクリック
2. 右上の「+ 新規登録」ボタンをクリック

登録画面：

```

│  道具の新規登録                         │
├─────────────────────────────────────────┤
│  基本情報                               │

│  │ カテゴリ *      ▼ 電動工具         │  │

│                                         │

│  ┌───────────────────────────────────┐  │

│                                         │

│  ┌───────────────────────────────────┐  │

│                                         │
│        [キャンセル]  [登録する]         │
└─────────────────────────────────────────┘
```

#### 重要なポイント

- **管理タイプ**:
  - **個別管理**: 1つ1つにシリアル番号が付く（ドリル、サンダーなど）
  - **消耗品**: 数量で管理する（軍手、テープなど）

- **個別アイテム数**:
  - 「3台」と入力すると、自動で #001、#002、#003 のシリアル番号が付いた個別アイテムが作成されます
  - それぞれに固有のQRコードが生成されます

- **保証期限**:
  - 設定すると、期限切れ前に自動で通知されます

- **詳細**: クリックすると詳細画面へ

## 📍 倉庫位置管理

### 倉庫階層の設定

道具を倉庫内のどこに保管しているか管理する機能です。

1. サイドバーの「設定・管理 ▼」→「📍 倉庫位置管理」をクリック
2. 右上の「⚙️ 階層設定」ボタンをクリック

```
┌─────────────────────────────────────────┐
│  倉庫階層の設定                         │
├─────────────────────────────────────────┤
│  倉庫内の位置をどう管理しますか？       │
│                                         │
│  階層1（例: エリア）                    │
│  ┌───────────────────────────────────┐  │
│  │ エリア                             │  │
│  └───────────────────────────────────┘  │
│                                         │
│  階層2（例: ラック）                    │
│  ┌───────────────────────────────────┐  │
│  │ ラック                             │  │
│  └───────────────────────────────────┘  │
│                                         │
│  階層3（例: 段）                        │
│  ┌───────────────────────────────────┐  │
│  │ 段                                 │  │
│  └───────────────────────────────────┘  │
│                                         │
│  位置コード形式: エリア-ラック-段        │
│  例: A-01-3 (Aエリアの1番ラックの3段目)  │
│                                         │
│        [キャンセル]  [保存する]         │
└─────────────────────────────────────────┘
```

### 倉庫位置を登録する

階層設定が完了したら、具体的な位置を登録します：

1. 「+ 新規登録」ボタンをクリック

```
┌─────────────────────────────────────────┐
│  倉庫位置の登録                         │
├─────────────────────────────────────────┤
│  位置コード *                           │
│  ┌───────────────────────────────────┐  │
│  │ A-01-3                             │  │
│  └───────────────────────────────────┘  │
│                                         │
│  表示名 *                               │
│  ┌───────────────────────────────────┐  │
│  │ Aエリア 1番ラック 3段目            │  │
│  └───────────────────────────────────┘  │
│                                         │
│  説明（任意）                           │
│  ┌───────────────────────────────────┐  │
│  │ 電動工具専用エリア                 │  │
│  └───────────────────────────────────┘  │
│                                         │
│  QRコード                               │
│  ☑ この位置のQRコードを生成する         │
│                                         │
│        [キャンセル]  [登録する]         │
└─────────────────────────────────────────┘
```

**QRコード生成のメリット**:
- 倉庫内の位置にもQRコードを貼れます
- スタッフが道具を返却するとき、位置のQRをスキャンするだけで正確な場所を記録できます

## 🔔 通知の管理

ヘッダー右上の🔔アイコンをクリックすると通知一覧が表示されます。

### 通知の種類

- **在庫アラート**: 消耗品が最小在庫を下回った
- **保証期限**: 道具の保証期限が近づいた（30日前・7日前）
- **システム通知**: 重要なお知らせ

### メール通知

重要な通知は自動でメール送信されます：
- 在庫不足アラート → 管理者にメール
- 保証期限切れ通知 → 管理者にメール

メール送信先は「組織設定」で変更できます。

## 💡 管理者向けTips

### 効率的な運用のコツ

1. **初期セットアップは丁寧に**
   - カテゴリを整理しておくと後が楽
   - 倉庫階層は実際の倉庫レイアウトに合わせる

2. **QRコードを活用**
   - 道具だけでなく、倉庫位置にもQRコードを貼る
   - A4用紙に複数のQRを印刷してラミネート加工すると耐久性UP

3. **定期的な棚卸**
   - 月1回、実際の在庫と システムの数を照合
   - 消耗品の「在庫調整」機能で修正

4. **スタッフ教育**
   - 移動登録を必ず行うよう徹底
   - 「一括移動」機能の使い方を実演

5. **通知設定の活用**
   - 最小在庫数を適切に設定
   - メール通知先を複数人に設定（CCで対応）

### よくある管理課題と解決策

| 課題 | 解決策 |
|-----|-------|
| スタッフが移動登録を忘れる | 朝礼で「一括移動」の重要性を説明。QRコードを大きく印刷 |
| 在庫数が合わない | 定期棚卸を実施。監査ログで不正を確認 |
| 道具がどこにあるか分からない | 倉庫位置管理を徹底。位置QRコードを活用 |
| 保証期限が切れて気づかない | 通知機能を有効化。メール送信先を確認 |

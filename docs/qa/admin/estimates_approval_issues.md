## 基本的な質問

### Q1. 見積書の承認フローとは何ですか？

**A.** 見積書の承認フローは、Leaderが作成した見積書をManager/Adminが承認し、顧客に送付するまでの一連の流れです。

**基本的な流れ**:
```
1. Leader が見積書を作成（下書き）
2. Leader が見積書を提出（提出済み）
3. Manager/Admin が内容を確認
4. Manager/Admin が承認（承認済み）
5. Manager/Admin または Leader が顧客に送付（顧客送付済）
6. Manager/Admin が顧客の判断を記録（顧客承認 or 顧客却下）
```

このフローにより、見積書の品質を担保し、適正な金額で顧客に提示できます。

### Q2. 誰が見積書を提出できますか？

**A.** **Leader、Manager、Admin**が見積書を提出できます。Staffは提出できません。

| 権限 | 提出 |
|------|------|
| **Staff** | ❌ |
| **Leader** | ✅ |
| **Manager** | ✅ |
| **Admin** | ✅ |

ただし、通常は**Leader**が見積書を作成・提出し、**Manager/Admin**が承認する運用が一般的です。

### Q3. 誰が見積書を承認できますか？

**A.** **Manager**または**Admin**のみが見積書を承認できます。Leaderは承認できません。

| 権限 | 承認 |
|------|------|
| **Staff** | ❌ |
| **Leader** | ❌ |
| **Manager** | ✅ |
| **Admin** | ✅ |

これにより、適切な権限者による承認プロセスが保証されます。

### Q4. 見積書のステータスにはどのような種類がありますか？

**A.** 見積書には以下の7つのステータスがあります：

| ステータス | 色 | 意味 |
|-----------|-----|------|
| **下書き** | 灰色 | 作成中・差し戻し後 |
| **提出済み** | オレンジ色 | 承認待ち |
| **提出済み（承認済み）** | オレンジ色（✓付き） | 承認済み・送付待ち |
| **顧客送付済** | 青色 | 顧客に送付済み・判断待ち |
| **顧客承認** | 緑色 | 受注確定 |
| **顧客却下** | 赤色 | 失注 |
| **期限切れ** | 黄色 | 有効期限切れ |

各ステータスで実行できる操作が異なります。

### Q5. 提出済みの見積書を編集できますか？

**A.** いいえ、**提出済みの見積書は編集できません**。編集するには以下の手順が必要です：

**方法1: Manager/Adminに差し戻しを依頼**
1. Manager/Adminに連絡
2. Manager/Adminが「差し戻し」を実行
3. ステータスが「下書き」に戻る
4. 編集可能になる

**方法2: 新規作成**
- 既存の見積書を参考に、新しい見積書を作成

**予防策**: 提出前に内容を十分に確認してください。

## 提出に関する質問（Leader）

### Q6. 見積書を提出する方法は？

**A.** 以下の手順で提出します：

1. 見積書一覧から見積書をクリック
2. 見積書詳細画面を開く
3. **「編集」**ボタンをクリック
4. 見積書編集画面の下部にある**「提出する」**ボタン（青色）をクリック
5. ステータスが「提出済み」に変更される

**注意**: 一度提出すると、Manager/Adminが差し戻すまで編集できません。

### Q7. 提出前に確認すべきポイントは？

**A.** 以下のポイントを確認してください：

**基本情報**:
- ✅ 顧客名が正しいか
- ✅ 件名が適切か
- ✅ 見積日が正しいか
- ✅ 有効期限が適切か（通常30日）

**明細**:
- ✅ 品目名・数量・単価が正しいか
- ✅ 金額が正しいか（自動計算）
- ✅ 税率が正しいか（通常10%）
- ✅ 不要な明細がないか

**合計金額**:
- ✅ 小計・消費税・合計が正しいか
- ✅ 予算内に収まっているか

### Q8. 提出後、承認されるまでどのくらいかかりますか？

**A.** 組織の運用によりますが、一般的には**1〜3営業日**程度です。

**確認方法**:
- 見積書詳細画面で「提出済み」ステータスを確認
- Manager/Adminが既読にすると、通知バッジが消えます

**催促が必要な場合**:
- 3営業日を過ぎても承認されない場合は、Manager/Adminに直接連絡してください

### Q9. 提出した見積書を取り下げることはできますか？

**A.** いいえ、提出した見積書を**自分で取り下げることはできません**。

**対応方法**:
1. Manager/Adminに連絡
2. 「差し戻し」を依頼
3. ステータスが「下書き」に戻る
4. 必要に応じて削除または修正

### Q10. 提出後、承認前に誤りを発見しました。どうすればよいですか？

**A.** すぐにManager/Adminに連絡してください。

**対応フロー**:
1. **Manager/Adminに連絡**（メール、チャット、口頭）
   - 「見積書#EST-20260121-0001に誤りがあります」
   - 「○○を○○に修正する必要があります」

2. **Manager/Adminが差し戻し**
   - 差し戻し理由: 「作成者から修正依頼あり」

3. **修正して再提出**
   - 誤りを修正
   - 再度「提出する」をクリック

## 承認に関する質問（Manager/Admin）

### Q11. 提出済みの見積書はどこで確認できますか？

**A.** 「見積管理」画面で、**未読の提出済み見積書**が赤い通知バッジ🔴で表示されます。

**表示例**:
```
見積書一覧
[提出済み] 見積書#EST-20260121-0001 🔴 未読
[提出済み] 見積書#EST-20260121-0002 （既読）
[下書き] 見積書#EST-20260121-0003
```

**既読にする方法**:
- 見積書詳細画面を開くと、自動的に既読になります

### Q12. 承認する際に確認すべきポイントは？

**A.** 以下のポイントを確認してください：

**金額の確認**:
- ✅ 単価が適正か（市場価格と比較）
- ✅ 数量が適切か
- ✅ 合計金額が予算内か
- ✅ 利益率が適切か

**内容の確認**:
- ✅ 顧客名が正しいか
- ✅ 件名が適切か
- ✅ 明細に不要な項目がないか
- ✅ 有効期限が適切か

**契約条件の確認**:
- ✅ 支払条件が適切か
- ✅ 納期が現実的か
- ✅ 特記事項が適切か

### Q13. 承認メモは必須ですか？

**A.** いいえ、承認メモは**任意**です。ただし、以下の場合は記載を推奨します：

**記載を推奨する場合**:
- 金額が高額（例: 100万円以上）
- 特殊な条件がある
- 後で確認が必要な内容がある

**記載例**:
```
承認メモ:
- 金額・内容とも問題なし
- 予算内に収まっている
- 顧客との協議済み
```

### Q14. 承認後、間違いに気づきました。どうすればよいですか？

**A.** 以下の対応を取ってください：

**顧客に送付する前**:
1. **「承認取り消し」**ボタンをクリック
2. **「差し戻し」**で下書きに戻す
3. Leaderに修正を依頼
4. 修正後、再度提出・承認

**顧客に送付した後**:
- 顧客送付済み（sent）の見積書は承認取り消しできません
- 新しい見積書を作成し直す必要があります

**重大な誤りの場合**:
- すぐに顧客に連絡
- 修正版の見積書を再送

### Q15. 複数のLeaderの見積書を同時に承認できますか？

**A.** いいえ、見積書は**1つずつ承認**する必要があります。

**承認手順**（複数ある場合）:
1. 見積書Aを開く → 承認
2. 見積書Bを開く → 承認
3. 見積書Cを開く → 承認

一括承認機能はありません。各見積書の内容を個別に確認して承認してください。

## 差し戻しに関する質問（Manager/Admin）

### Q16. どのような場合に差し戻すべきですか？

**A.** 以下のような場合に差し戻してください：

**金額の問題**:
- 単価が高すぎる
- 合計金額が予算オーバー
- 利益率が低すぎる

**内容の問題**:
- 明細に不要な項目がある
- 数量が多すぎる/少なすぎる
- 件名が不明確

**形式の問題**:
- 顧客名が間違っている
- 有効期限が不適切
- 特記事項が欠けている

**その他**:
- Leader本人から修正依頼があった

### Q17. 差し戻し理由は必須ですか？

**A.** はい、差し戻し理由は**必須**です。入力しないと差し戻しできません。

**理由**:
- Leaderが何を修正すべきか明確にするため
- 修正の方向性を示すため
- コミュニケーションを円滑にするため

**良い差し戻し理由の例**:
```
✅ 具体的:
「品目Aの単価を5,000円から4,500円に変更してください」

✅ 理由付き:
「合計金額が予算の50万円を超えているため、10%削減してください」

✅ 代替案提示:
「品目Bは不要と思われます。削除して再計算してください」
```

**悪い差し戻し理由の例**:
```
❌ 不明確:
「修正してください」

❌ 理由なし:
「ダメです」

❌ 感情的:
「何を考えているんですか」
```

### Q18. 差し戻し後、Leaderはどうすればよいですか？

**A.** Leaderは以下の手順で対応します：

1. **差し戻し理由を確認**
   - 見積書詳細画面に差し戻し理由が表示されます

2. **見積書を修正**
   - 「編集」ボタンをクリック
   - 差し戻し理由に従って修正

3. **再提出**
   - 修正後、「提出する」ボタンをクリック

4. **Manager/Adminが再承認**
   - 修正内容を確認して承認

### Q19. 差し戻し回数に制限はありますか？

**A.** いいえ、差し戻し回数に制限はありません。何度でも差し戻し・再提出が可能です。

**ただし**:
- 頻繁な差し戻しは業務効率を下げます
- 1回目の差し戻しで明確な指示を出すことを推奨します
- 3回以上差し戻す場合は、口頭で直接説明することを推奨します

### Q20. 差し戻しをキャンセルできますか？

**A.** いいえ、差し戻しを**キャンセルすることはできません**。

**差し戻し後の状態**:
- ステータス: 「下書き」
- 差し戻し理由が記録される

**対応方法**:
- Leaderに「修正不要」と連絡
- Leaderが内容を確認して再提出
- 再度承認

## 顧客送付に関する質問

### Q21. 「送付済みにする」ボタンを押すと、自動でメール送信されますか？

**A.** いいえ、**自動でメール送信されません**。このボタンは、実際に顧客へ見積書を送付した後に、システム上のステータスを「顧客送付済」に変更するためのものです。

**正しい使い方**:
1. **手動で顧客に見積書を送付**（メール添付、郵送、FAXなど）
2. **システムで「送付済みにする」ボタンをクリック**
3. ステータスが「顧客送付済」に変更される

### Q22. 顧客に見積書を送付する方法は？

**A.** 以下の方法で送付できます：

**1. メール添付（推奨）**:
- 見積書詳細画面で「PDF出力」ボタンをクリック
- PDFをダウンロード
- メールに添付して送信

**2. 郵送**:
- 見積書を印刷
- 封筒に入れて郵送

**3. FAX**:
- 見積書を印刷
- FAXで送信

**4. 手渡し**:
- 見積書を印刷
- 顧客に直接手渡し

いずれの方法でも、送付後に「送付済みにする」ボタンをクリックしてください。

### Q23. Leaderは自分が作成した見積書を送付できますか？

**A.** はい、Leaderは**自分が作成した見積書のみ**送付できます（承認済みの場合）。

| 権限 | 自分が作成した見積書 | 他人が作成した見積書 |
|------|---------------------|---------------------|
| **Leader** | ✅ 送付可能 | ❌ 送付不可 |
| **Manager** | ✅ 送付可能 | ✅ 送付可能 |
| **Admin** | ✅ 送付可能 | ✅ 送付可能 |

### Q24. 送付済みの見積書を再度送付することはできますか？

**A.** システム上のステータス変更はできませんが、**PDFを再ダウンロードして再送付**することは可能です。

**手順**:
1. 見積書詳細画面を開く
2. 「PDF出力」ボタンをクリック
3. PDFをダウンロード
4. 顧客にメールで再送

**用途**:
- 顧客が見積書を紛失した
- 別の担当者にも送付したい
- 印刷版が必要になった

### Q25. 承認前の見積書を顧客に送付してしまいました。どうすればよいですか？

**A.** 以下の対応を取ってください：

**ステップ1: すぐにManager/Adminに報告**
- 承認前の見積書を送付したことを報告
- 緊急対応を依頼

**ステップ2: Manager/Adminが至急承認**
- 内容を確認して承認
- または、差し戻して修正

**ステップ3: 顧客に連絡**
- 内容に問題がない場合: そのまま
- 内容に問題がある場合: 修正版を再送

**予防策**: 承認済みであることを必ず確認してから送付してください。

## 顧客判断記録に関する質問

### Q26. 「顧客承認」と「顧客却下」の違いは何ですか？

**A.** 以下の違いがあります：

| 項目 | 顧客承認 | 顧客却下 |
|------|---------|---------|
| **意味** | 顧客が見積書を承認 | 顧客が見積書を却下 |
| **ステータス色** | 緑色 | 赤色 |
| **次のアクション** | 請求書作成 | なし（案件終了） |
| **受注** | 受注確定 | 失注 |

### Q27. 顧客承認を記録すると、自動で請求書が作成されますか？

**A.** いいえ、**自動で請求書は作成されません**。顧客承認を記録した後、手動で請求書を作成する必要があります。

**請求書作成手順**:
1. 見積書詳細画面を開く
2. 「請求書作成」ボタンをクリック
3. 請求書作成画面で内容を確認
4. 請求書を登録

詳細は[請求書作成マニュアル](/manual/invoices_create)を参照してください。

### Q28. 顧客から「検討中」と言われました。どのステータスにすればよいですか？

**A.** **「顧客送付済」のまま**にしてください。顧客の最終判断を待ちます。

**ステータスの使い分け**:
- **顧客送付済**: 顧客が検討中（判断待ち）
- **顧客承認**: 顧客が「承認します」と明確に回答
- **顧客却下**: 顧客が「お断りします」と明確に回答

顧客から明確な回答があるまで、「顧客送付済」ステータスを維持してください。

### Q29. 顧客却下を記録すると、見積書は削除されますか？

**A.** いいえ、**見積書は削除されません**。ステータスが「顧客却下」（赤色）に変更されるだけです。

**顧客却下後の状態**:
- ✅ 見積書詳細を閲覧可能
- ✅ PDF出力可能
- ❌ 編集不可
- ❌ 削除不可

**用途**:
- 過去の見積履歴として保存
- 次回の見積作成時の参考資料

### Q30. 顧客承認後、顧客却下に変更できますか？

**A.** いいえ、システム上は**顧客承認から顧客却下に変更する機能はありません**。

**対応方法**:
1. Adminに連絡
2. データベースで直接修正（管理者のみ）
3. または、新しい見積書を作成し直す

**予防策**: 顧客から明確な回答を確認してから記録してください。

## 期限切れに関する質問

### Q31. 見積書が期限切れになる条件は？

**A.** 見積書の**有効期限を過ぎる**と、自動的に「期限切れ」ステータスになります。

**例**:
```
見積日: 2026-01-21
有効期限: 2026-02-20
↓
2026-02-21になると自動的に期限切れ
```

**すべてのステータスが対象**:
- 下書き
- 提出済み
- 顧客送付済
- 顧客承認・却下以外

### Q32. 期限切れの見積書は編集できますか？

**A.** いいえ、**期限切れの見積書は編集できません**。

**期限切れ後の制限**:
- ❌ 編集不可
- ❌ 承認不可
- ❌ 差し戻し不可
- ❌ 顧客送付不可
- ✅ 閲覧のみ可能

**対応方法**:
- 新しい見積書を作成し直す
- 有効期限を延長した見積書を複製して作成

### Q33. 期限切れを防ぐ方法は？

**A.** 以下の方法で期限切れを防げます：

**1. 有効期限を長めに設定**
- 通常: 30日
- 余裕を持たせる: 60日または90日

**2. 早めに提出・承認**
- Leaderは作成後すぐに提出
- Manager/Adminは3営業日以内に承認

**3. 有効期限を確認**
- 見積書詳細画面で有効期限を定期的に確認
- 期限が近い見積書は優先的に処理

**4. リマインダー設定**
- カレンダーに有効期限を登録
- 期限の1週間前にリマインダー

### Q34. 期限切れの見積書を復活させることはできますか？

**A.** いいえ、**期限切れの見積書を復活させることはできません**。

**対応方法**:
1. 期限切れの見積書を参考に、新しい見積書を作成
2. 有効期限を新しい日付に設定
3. 内容を確認して提出・承認

**データ引き継ぎ**:
- 顧客、プロジェクト、件名、明細などを手動でコピー
- または、見積書画面を見ながら同じ内容を入力

### Q35. 顧客承認済みの見積書も期限切れになりますか？

**A.** いいえ、**顧客承認済み**および**顧客却下**の見積書は期限切れになりません。

**期限切れになるステータス**:
- ✅ 下書き
- ✅ 提出済み
- ✅ 顧客送付済

**期限切れにならないステータス**:
- ❌ 顧客承認（受注確定のため）
- ❌ 顧客却下（既に終了のため）

顧客が承認した見積書は、有効期限に関係なく有効です。

## トラブルシューティング

### Q36. 「承認する」ボタンが表示されません

**A.** 以下を確認してください：

**1. 権限を確認**
- 自分の権限が Manager または Admin か確認
- Leader は承認できません

**2. ステータスを確認**
- ステータスが「提出済み」か確認
- 下書きや顧客送付済みは承認できません

**3. 既に承認済みか確認**
- 承認済みの場合は「承認取り消し」ボタンが表示されます

### Q37. 「提出する」ボタンが表示されません

**A.** 以下を確認してください：

**1. 権限を確認**
- 自分の権限が Leader / Manager / Admin か確認
- Staff は提出できません

**2. 編集画面を開いているか確認**
- 見積書詳細画面には「提出する」ボタンはありません
- 「編集」ボタンをクリックして編集画面を開いてください

**3. ステータスを確認**
- ステータスが「下書き」か確認
- 提出済み以降は「提出する」ボタンは表示されません

### Q38. 差し戻しボタンを押してもエラーになります

**A.** 以下を確認してください：

**1. 差し戻し理由を入力したか確認**
- 差し戻し理由は必須です
- 空欄の場合は「差し戻し理由を入力してください」とエラーが表示されます

**2. ステータスを確認**
- ステータスが「提出済み」か確認
- 承認済み・送付済みは差し戻しできません

**3. 権限を確認**
- 自分の権限が Manager または Admin か確認

### Q39. PDF出力ボタンが表示されません

**A.** 以下を確認してください：

**1. 権限を確認**
- PDF出力は Manager / Admin のみ可能
- Leader / Staff は表示されません

**2. 承認状態を確認**
- 承認済みの見積書のみPDF出力可能
- 未承認の場合は表示されません

**3. ステータスを確認**
- 提出済み（承認済み）、顧客送付済み、顧客承認、顧客却下のみ
- 下書き、提出済み（未承認）は表示されません

### Q40. 見積書一覧に未読バッジが表示されません

**A.** 以下を確認してください：

**1. 権限を確認**
- 未読バッジは Manager / Admin のみ表示
- Leader / Staff には表示されません

**2. 既読済みか確認**
- 一度開いた見積書は既読になります
- 未読の見積書のみバッジが表示されます

**3. ステータスを確認**
- 「提出済み」ステータスのみバッジが表示されます
- 下書きや送付済みにはバッジは表示されません

# 出退勤設定 Q&A

## 基本

### Q1: 出退勤設定とは何ですか？

**A**: 出退勤管理（勤怠管理）に関する組織全体の設定を変更する機能です。打刻方法、休憩時間の記録方式、リマインダー通知、管理者向けレポート・アラートなどを設定できます。設定は組織全体のスタッフに適用されます。

### Q2: 誰が出退勤設定にアクセスできますか？

**A**: 管理者（Admin）とマネージャー（Manager）がアクセスできます。リーダーおよび一般スタッフは設定ページにアクセスできません。権限が必要な場合は、管理者に権限変更を依頼してください。

### Q3: 出退勤設定ページにはどうやってアクセスしますか？

**A**: 以下の2つの方法でアクセスできます：
1. ダッシュボード → 「設定」 → 「出退勤設定」
2. サイドメニュー → 「設定」 → 「出退勤設定」

### Q4: 設定を変更したら即座に全スタッフに反映されますか？

**A**: はい、設定を保存すると即座に組織全体に反映されます。例えば、QRコード打刻を有効にすると、すべてのスタッフがQRコードで打刻できるようになります。ただし、スタッフがアプリをリフレッシュする必要がある場合があります。

### Q5: 出退勤設定を誤って変更してしまいました。元に戻せますか？

**A**: 設定変更の履歴は自動保存されません。元に戻すには、手動で以前の設定値に変更してください。重要な設定を変更する前に、現在の設定をスクリーンショットで記録しておくことを推奨します。

## 打刻方法設定

### Q6: 打刻方法とは何ですか？

**A**: スタッフが出退勤を記録する方法です。2つの方法があります：

1. **手動打刻（ボタンタップ）**: 場所を選択して打刻ボタンをタップ
2. **QRコード打刻**: QRコードをスキャンして場所を自動判別

複数選択可能です。

### Q7: 手動打刻とQRコード打刻のどちらを有効にすべきですか？

**A**: 最初は手動打刻のみを有効にすることを推奨します。運用に慣れたら、固定場所での打刻を厳密に管理したい場合にQRコード打刻を追加してください。

**推奨設定**：
- 初期: 手動打刻のみ
- 厳密管理: 手動打刻 + QRコード打刻

### Q8: 打刻方法を両方とも無効にできますか？

**A**: いいえ、少なくとも1つの打刻方法を有効にする必要があります。すべて無効にすると、スタッフが出退勤を記録できなくなります。

### Q9: QRコード打刻を有効にすると、手動打刻は使えなくなりますか？

**A**: いいえ、両方とも使えます。QRコード打刻を有効にしても、手動打刻も引き続き利用できます。スタッフは打刻方法を選択できます。

### Q10: QRコード更新頻度はどれを選ぶべきですか？

**A**: 7日（週1回更新）を推奨します。セキュリティと運用負荷のバランスが良いです。

| 更新頻度 | 推奨シーン |
|---------|-----------|
| 1日 | 最高セキュリティが必要 |
| 3日 | セキュリティ重視 |
| **7日（推奨）** | 多くの組織に最適 |
| 30日 | QR掲示替えが困難 |

## 休憩時間設定

### Q11: 休憩時間モードの違いは何ですか？

**A**: 3つのモードから選択できます：

| モード | 説明 | 使用シーン |
|--------|------|-----------|
| **記録しない** | 休憩時間は記録せず、勤務時間のみ管理 | 休憩記録が不要 |
| **簡易モード** | 1回の休憩のみ記録（開始・終了ボタン） | 昼休憩のみ記録 |
| **詳細モード** | 複数回の休憩を記録可能 | 複数回の休憩を正確に記録 |

**推奨**: 簡易モード

### Q12: 休憩時間を自動差し引きとは何ですか？

**A**: スタッフが休憩開始・終了ボタンを押さなくても、指定した時間を自動で差し引く機能です。休憩時間が固定の組織（例：毎日45分の昼休憩）に便利です。

**例**：
- 出勤: 9:00
- 退勤: 18:00
- 自動差し引き: 45分
- → 実働時間: 8時間15分（9時間から45分を自動差し引き）

### Q13: 自動休憩差し引きを有効にすると、休憩開始・終了ボタンは使えなくなりますか？

**A**: いいえ、使えます。自動差し引きを有効にしても、スタッフは任意で休憩開始・終了ボタンを使用できます。ただし、手動で記録した休憩時間が優先されます。

### Q14: 自動差し引き時間は何分に設定すべきですか？

**A**: 組織の休憩時間に合わせて設定してください。

**一般的な設定**：
- 45分: 一般的な昼休憩
- 60分: 1時間の昼休憩
- 90分: 1時間30分の昼休憩

15分単位で設定することを推奨します。

### Q15: 休憩時間モードを「記録しない」にすると、勤務時間はどうなりますか？

**A**: 出勤時刻から退勤時刻までの時間が勤務時間として記録されます。休憩時間は差し引かれません。

**例**：
- 出勤: 9:00
- 退勤: 18:00
- → 勤務時間: 9時間（休憩時間を含む）

## リマインダー設定

### Q16: 出勤リマインダーとは何ですか？

**A**: 指定した時刻に、まだ出勤打刻していないスタッフに通知を送信する機能です。打刻忘れを防ぐことができます。

**例**：
- 設定時刻: 10:00
- 10:00の時点で出勤打刻していないスタッフに通知

### Q17: 出勤リマインダーの時刻はいつに設定すべきですか？

**A**: 始業時刻の30分〜1時間後を推奨します。

**例**：
- 始業時刻が9:00 → 10:00に設定
- 始業時刻が8:00 → 9:00に設定
- 始業時刻が10:00 → 11:00に設定

### Q18: 退勤リマインダーとは何ですか？

**A**: 指定した時刻に、まだ退勤打刻していないスタッフに通知を送信する機能です。退勤打刻忘れを防ぎ、長時間労働を防止します。

**例**：
- 設定時刻: 20:00
- 20:00の時点で退勤打刻していないスタッフに通知

### Q19: 退勤リマインダーの時刻はいつに設定すべきですか？

**A**: 定時終業時刻の1〜2時間後を推奨します。

**例**：
- 定時が18:00 → 20:00に設定
- 定時が17:00 → 19:00に設定
- 定時が19:00 → 21:00に設定

### Q20: リマインダーは誰に送信されますか？

**A**: 対象となるスタッフ（出勤または退勤打刻を忘れているスタッフ）に送信されます。管理者には送信されません。

## 管理者向け設定

### Q21: 日次レポートとは何ですか？

**A**: 毎日指定した時刻に、出退勤状況のレポートを管理者とマネージャーに送信する機能です。

**レポート内容**：
- 前日の出退勤状況
- 打刻忘れのスタッフ一覧
- 長時間労働のスタッフ一覧

### Q22: 日次レポートの送信時刻はいつに設定すべきですか？

**A**: 始業時刻の30分〜1時間後を推奨します。

**例**：
- 始業時刻が9:00 → 10:00に設定
- 始業時刻が8:00 → 9:00に設定

理由：始業後に前日の状況を確認できるため。

### Q23: 日次レポートはどこに送信されますか？

**A**: システム内の通知センターに表示されます。「メールで送信」を有効にすると、管理者とマネージャーのメールアドレスにも送信されます。

### Q24: 日次レポートのメール送信は有効にすべきですか？

**A**: はい、推奨します。メールで確認できると便利です。システムにログインしなくても状況を把握できます。

### Q25: 長時間労働アラートとは何ですか？

**A**: 指定した時間を超えて勤務しているスタッフに対して、管理者に通知を送信する機能です。長時間労働を防ぐことができます。

**例**：
- アラート基準時間: 12時間
- スタッフが12時間を超えて勤務 → 管理者に通知

### Q26: 長時間労働アラートの基準時間は何時間に設定すべきですか？

**A**: 組織の実態に合わせて設定してください。

**一般的な設定**：
- 10時間: 残業2時間を超えたら通知（8時間勤務の場合）
- 12時間（推奨）: 残業を含めて12時間を超えたら通知
- 15時間: 深夜勤務が発生したら通知

### Q27: 長時間労働アラートは誰に送信されますか？

**A**: 管理者とマネージャーに送信されます。対象となるスタッフ本人には送信されません。

### Q28: 日次レポートと長時間労働アラートは必須ですか？

**A**: 必須ではありませんが、推奨します。

**推奨理由**：
- 日次レポート: 出退勤状況を把握し、打刻忘れを早期発見
- 長時間労働アラート: 長時間労働を防ぎ、スタッフの健康を守る

### Q29: 日次レポートが送信されません。なぜですか？

**A**: 以下を確認してください：

1. 日次レポートが有効（チェックボックスがON）になっているか
2. 送信時刻が正しく入力されているか（HH:MM形式）
3. 管理者・マネージャーのメールアドレスが登録されているか（メール送信を有効にしている場合）

### Q30: 長時間労働アラートが送信されません。なぜですか？

**A**: 以下を確認してください：

1. 長時間労働アラートが有効（チェックボックスがON）になっているか
2. アラート基準時間が正しく設定されているか（8〜24時間）
3. 実際に基準時間を超えて勤務しているスタッフがいるか

## 設定のベストプラクティス

### Q31: 新規導入時の推奨設定は何ですか？

**A**: 以下の設定を推奨します：

| 設定項目 | 推奨値 |
|---------|--------|
| 手動打刻 | ✅ 有効 |
| QRコード打刻 | ❌ 無効（慣れたら有効化） |
| 休憩時間モード | 簡易モード |
| 自動休憩差し引き | ❌ 無効（慣れたら有効化） |
| 出勤リマインダー | ✅ 有効（10:00） |
| 退勤リマインダー | ✅ 有効（20:00） |
| 日次レポート | ✅ 有効（10:00） |
| 長時間労働アラート | ✅ 有効（12時間） |

### Q32: 運用に慣れたら、どの設定を変更すべきですか？

**A**: 以下の変更を検討してください：

1. **QRコード打刻を有効化**: 固定場所での打刻を厳密に管理
2. **自動休憩差し引きを有効化**: 休憩時間が固定の場合、運用が楽になる
3. **休憩時間モードを詳細モードに変更**: 複数回の休憩を正確に記録
4. **長時間労働アラートの基準時間を調整**: 組織の実態に合わせて調整

### Q33: シンプル運用（小規模組織向け）の設定例は？

**A**: 以下の設定を推奨します：

```
打刻方法: 手動打刻のみ
休憩時間: 簡易モード + 自動差し引き45分
リマインダー: 出勤・退勤ともに有効
日次レポート: 有効
長時間労働アラート: 無効
```

**特徴**: 最小限の設定で運用開始。打刻忘れを防ぎつつ、シンプルな管理。

### Q34: 厳密管理（中規模〜大規模組織向け）の設定例は？

**A**: 以下の設定を推奨します：

```
打刻方法: 手動打刻 + QRコード打刻（7日更新）
休憩時間: 詳細モード（自動差し引きなし）
リマインダー: 出勤・退勤ともに有効
日次レポート: 有効 + メール送信
長時間労働アラート: 有効（12時間）
```

**特徴**: QRコード打刻で場所を厳密に管理。複数回の休憩を正確に記録。長時間労働を監視。

### Q35: 完全自動化（運用負荷最小）の設定例は？

**A**: 以下の設定を推奨します：

```
打刻方法: 手動打刻のみ
休憩時間: 簡易モード + 自動差し引き60分
リマインダー: 出勤・退勤ともに有効
日次レポート: 有効
長時間労働アラート: 有効（10時間）
```

**特徴**: 休憩時間を自動差し引きで管理。リマインダーとレポートで自動化。

## トラブルシューティング

### Q36: 設定ページにアクセスできません。

**A**: リーダーまたは一般スタッフでログインしている可能性があります。出退勤設定ページは管理者とマネージャーのみがアクセスできます。権限が必要な場合は、管理者に権限変更を依頼してください。

### Q37: 設定を保存できません。エラーが表示されます。

**A**: 以下を確認してください：

1. すべての時刻が正しく入力されているか（HH:MM形式）
2. 自動差し引き時間が0〜120分の範囲内か
3. 長時間労働アラートの基準時間が8〜24時間の範囲内か
4. ネットワーク接続を確認
5. ブラウザをリフレッシュ（F5）して再度試す

### Q38: 設定変更後もスタッフに反映されません。

**A**: 以下を試してください：

1. スタッフにアプリをリフレッシュしてもらう（アプリを閉じて再起動）
2. ブラウザ版の場合は、ブラウザをリフレッシュ（F5）してもらう
3. 一度ログアウトして再ログインしてもらう

### Q39: リマインダー通知が送信されません。

**A**: 以下を確認してください：

1. リマインダーが有効（チェックボックスがON）になっているか
2. 時刻が正しく入力されているか（HH:MM形式）
3. スタッフの通知設定がオフになっていないか（ブラウザまたはアプリの通知設定）

### Q40: QRコードが読み取れません。

**A**: 以下を確認してください：

1. QRコード打刻が有効になっているか
2. QRコードが古い（有効期限切れ）場合は、再生成してください
3. QRコード更新頻度の設定に応じて、定期的に再生成してください

**QRコード再生成方法**：
- 管理者が「QRコード管理」ページから新しいQRコードを生成
- QR更新頻度が7日の場合、7日ごとに自動更新されます

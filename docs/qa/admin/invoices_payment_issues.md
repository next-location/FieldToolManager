# 入金記録 Q&A

## 基本

### Q1: 入金記録機能とは何ですか？

**A**: 請求書に対する顧客からの入金を記録する機能です。入金日、入金額、支払方法などを記録し、請求書の既入金額と残額を自動更新します。全額入金が完了すると、請求書のステータスが自動的に「支払済み」になります。

### Q2: 誰が入金記録を登録できますか？

**A**: リーダー（Leader）以上が入金記録を登録できます。一般スタッフは入金記録機能にアクセスできません。権限が不足している場合は、管理者に権限変更を依頼してください。

### Q3: 入金記録はどこから登録しますか？

**A**: 2つの方法があります：
1. **請求書詳細ページから**: 請求書詳細ページの右上にある「入金登録」ボタンをクリック（ステータスが「送付済み」で未払いがある場合のみ表示）
2. **入金一覧ページから**: サイドメニュー「入金・支払」→「新規登録」→支払タイプで「入金」を選択

### Q4: 入金記録の必須項目は何ですか？

**A**: 以下の4項目が必須です：
- **入金日**: 実際に入金があった日付
- **請求書**: 入金対象の請求書
- **入金額**: 今回入金された金額
- **支払方法**: 銀行振込、現金、小切手、クレジットカード、その他から選択

参照番号とメモは任意項目です。

### Q5: 入金記録を編集・削除できますか？

**A**: いいえ、入金記録は登録後の編集・削除ができません。誤って登録した場合は、管理者またはシステム管理者に連絡してデータベースから手動で修正を依頼してください。登録前に必ず金額と日付を確認してください。

## 入金額と計算

### Q6: 部分入金とは何ですか？

**A**: 請求金額の一部だけを入金することです。例えば、¥1,000,000の請求書に対して¥300,000だけ入金する場合などです。部分入金の場合、請求書のステータスは「送付済み」のままで、残額が表示されます。複数回に分けて入金記録を登録できます。

### Q7: 入金後残額はどのように計算されますか？

**A**: 以下の計算式で自動計算されます：

```
入金後残額 = 請求金額 - 既入金額 - 今回の入金額
```

**例**：
- 請求金額: ¥1,000,000
- 既入金額: ¥300,000
- 今回の入金額: ¥500,000
- → 入金後残額: ¥200,000

入金額を入力すると、入金後残額がリアルタイムで表示されます。

### Q8: 請求金額を超える金額は入力できますか？

**A**: いいえ、残額を超える金額は入力できません。フォームで入力しようとするとエラーメッセージが表示されます。過入金（overpayment）が発生した場合は、まず残額分を入力して請求書を「支払済み」にし、管理者に相談してください。

### Q9: 振込手数料が引かれて入金された場合はどうすればよいですか？

**A**: 実際に入金された金額を入金額として記録してください。

**例**：
- 請求金額: ¥100,000
- 振込手数料: ¥440
- 実際の入金額: ¥99,560
- → 入金額に「99560」と入力
- → メモ欄に「振込手数料440円引き」と記録

これにより、実際の入金額と帳簿が一致します。

### Q10: 複数回の部分入金を記録できますか？

**A**: はい、可能です。同じ請求書に対して何度でも入金記録を登録できます。既入金額は自動的に累積され、残額が再計算されます。

**例**：
- 1回目: ¥300,000入金 → 既入金額: ¥300,000、残額: ¥700,000
- 2回目: ¥500,000入金 → 既入金額: ¥800,000、残額: ¥200,000
- 3回目: ¥200,000入金 → 既入金額: ¥1,000,000、残額: ¥0、ステータス: 支払済み

## 支払方法

### Q11: どのような支払方法を選択できますか？

**A**: 以下の5種類から選択できます：

| 支払方法 | 説明 | 使用例 |
|----------|------|--------|
| **銀行振込** | 最も一般的な方法 | 銀行口座への振込 |
| **現金** | 現金での直接支払い | 対面での現金受取 |
| **小切手** | 小切手による支払い | 小切手の受領 |
| **クレジットカード** | クレジットカード決済 | カード決済システム経由 |
| **その他** | 上記以外の支払方法 | 手形、電子マネーなど |

### Q12: 参照番号とは何ですか？入力する必要がありますか？

**A**: 参照番号は任意項目で、入金を識別するための番号や情報を記録できます。

**例**：
- 銀行振込の場合: 振込依頼人名、取引番号
- クレジットカードの場合: 決済ID、トランザクションID
- その他: 管理用の識別番号

参照番号を記録しておくと、後で入金の照合作業がスムーズになりますが、必須ではありません。

### Q13: メモ欄には何を書けばよいですか？

**A**: 入金に関する補足情報を自由に記録できます。任意項目ですが、以下のような情報を記録すると便利です：

- 「請求書No.2024-001に対する2回目の入金」
- 「振込手数料440円引かれて入金」
- 「顧客から端数調整の了承を得て-50円」
- 「月末締め、翌月15日払いのスケジュール通り入金」
- 「担当者: 山田太郎様より現金受領」

### Q14: 支払方法で「その他」を選ぶのはどんな場合ですか？

**A**: 以下のような場合に「その他」を選択してください：

- 手形による支払い
- 電子マネー決済（PayPay、LINE Payなど）
- 相殺（取引先との債権・債務を相殺）
- バーター取引（物品やサービスとの交換）

「その他」を選んだ場合は、メモ欄に具体的な支払方法を記載することを推奨します。

### Q15: 銀行振込の場合、参照番号に何を入力すればよいですか？

**A**: 以下のような情報を入力することを推奨します：

- 振込依頼人名（顧客名と異なる場合）
- 銀行の取引番号・照合番号
- 振込日時
- 振込元の銀行名・支店名

**例**：「三菱UFJ銀行 新宿支店から振込、取引番号: 20240115-12345」

入金の照合時に役立ちます。

## ステータス変化

### Q16: 入金記録後、請求書のステータスはどうなりますか？

**A**: 入金額によって以下のように変化します：

| 入金パターン | ステータス変化 |
|-------------|---------------|
| **部分入金（残額あり）** | 「送付済み」のまま |
| **全額入金（残額0円）** | 「送付済み」→「支払済み」に自動変更 |

全額入金が完了すると、「入金登録」ボタンが非表示になります。

### Q17: 「入金登録」ボタンが表示されないのはなぜですか？

**A**: 以下のいずれかの理由が考えられます：

1. **権限不足**: 一般スタッフでログインしている → リーダー以上の権限を持つユーザーに入金記録を依頼してください
2. **請求書のステータスが不適切**: 「下書き」「承認待ち」の場合 → 請求書を「送付済み」にしてください
3. **すでに全額入金済み**: 既入金額 = 請求金額の場合 → 入金記録は完了しています

請求書のステータスと権限を確認してください。

### Q18: 部分入金の場合、請求書のステータスはいつまで「送付済み」ですか？

**A**: 残額が0円になるまで「送付済み」のままです。

**例**：
```
請求金額: ¥1,000,000
1回目入金: ¥300,000 → ステータス: 送付済み（残額: ¥700,000）
2回目入金: ¥500,000 → ステータス: 送付済み（残額: ¥200,000）
3回目入金: ¥200,000 → ステータス: 支払済み（残額: ¥0）
```

全額入金が完了した時点で自動的に「支払済み」に変更されます。

### Q19: 請求書が「支払済み」になった後も、入金記録を追加できますか？

**A**: いいえ、できません。請求書が「支払済み」になると「入金登録」ボタンが非表示になり、新たな入金記録を追加できなくなります。過入金があった場合や、返金処理が必要な場合は、管理者に相談してください。

### Q20: 入金記録を削除したら、請求書のステータスは戻りますか？

**A**: 入金記録は通常のUIから削除できません。もし管理者がデータベースから入金記録を手動で削除した場合、既入金額が再計算され、残額が復活します。全額入金だった場合は、ステータスが「支払済み」から「送付済み」に戻る可能性があります。

## 入金履歴と確認

### Q21: 入金履歴はどこで確認できますか？

**A**: 2つの場所で確認できます：

1. **請求書詳細ページ**: 「入金履歴」セクションにその請求書に対する全ての入金記録が表示されます
2. **入金一覧ページ**: サイドメニュー「入金・支払」をクリックすると、全ての入金記録が一覧表示されます

入金履歴には、入金日、入金額、支払方法、参照番号、メモ、登録者名、登録日時が表示されます。

### Q22: 入金履歴に表示される情報は何ですか？

**A**: 各入金記録について以下の情報が表示されます：

- **入金日**: 実際に入金があった日付
- **入金額**: 入金された金額
- **支払方法**: 銀行振込、現金など
- **参照番号**: 振込番号など
- **メモ**: 補足情報
- **登録者名**: 誰が入金記録を登録したか
- **登録日時**: 入金記録を登録した日時

### Q23: 入金一覧ページで請求書ごとにフィルタリングできますか？

**A**: はい、入金一覧ページでは請求書ごとのフィルタリングや検索が可能です。特定の請求書に対する入金記録だけを表示したい場合は、検索機能を使用してください。

### Q24: 入金記録は誰が登録したかわかりますか？

**A**: はい、各入金記録には「登録者名」が記録されます。入金履歴を確認すると、誰がいつ入金記録を登録したかがわかります。これにより、入金記録の責任者が明確になります。

### Q25: 入金履歴は印刷できますか？

**A**: 請求書詳細ページの入金履歴セクションを印刷することで、入金履歴を紙に出力できます。ブラウザの印刷機能（Ctrl+P または Cmd+P）を使用してください。

## 入金日

### Q26: 入金日にはどの日付を設定すればよいですか？

**A**: 実際に銀行口座に入金された日付を設定してください。振込日と入金日が異なる場合があります（土日祝日の場合など）。正確な入金日を記録することで、キャッシュフロー管理がスムーズになります。

### Q27: 入金日のデフォルトは何ですか？

**A**: 入金日のデフォルトは今日の日付です。実際の入金日と異なる場合は、カレンダーから正しい日付を選択してください。

### Q28: 過去の日付を入金日に設定できますか？

**A**: はい、可能です。入金の記録が遅れた場合など、過去の日付を入金日として設定できます。カレンダーから実際に入金があった日付を選択してください。

### Q29: 未来の日付を入金日に設定できますか？

**A**: 技術的には可能ですが、推奨されません。入金日は実際に入金があった日付を記録するためのものです。未来の入金予定を記録する場合は、実際に入金があってから入金記録を登録してください。

### Q30: 入金日と登録日時の違いは何ですか？

**A**:
- **入金日**: 実際に銀行口座に入金があった日付（ユーザーが入力）
- **登録日時**: 入金記録をシステムに登録した日時（自動記録）

通常は同じ日ですが、入金の記録が遅れた場合は異なる日付になることがあります。

## トラブルシューティング

### Q31: 誤った金額で入金記録を登録してしまいました。どうすればよいですか？

**A**: 入金記録は編集・削除できないため、以下の手順で対処してください：

1. 管理者またはシステム管理者に連絡してください
2. 誤った入金記録をデータベースから手動で削除または修正してもらってください
3. 必要に応じて、正しい金額で入金記録を再登録してください

今後のために、登録前に必ず金額を確認してください。

### Q32: 請求書を選択しても残額が表示されません。なぜですか？

**A**: 以下の原因が考えられます：

1. **データの読み込み遅延**: 少し待ってから画面をリフレッシュしてください
2. **請求書データの不整合**: 請求金額が設定されていない可能性があります → 請求書を確認してください
3. **ブラウザの問題**: ブラウザをリフレッシュ（F5）してください

問題が解決しない場合は、管理者に連絡してください。

### Q33: 入金額を入力するとエラーメッセージが表示されます。

**A**: 以下の原因が考えられます：

1. **残額を超えている**: 入力した金額が残額を超えています → 残額以下の金額を入力してください
2. **数字以外の文字を入力**: 数字のみを入力してください（カンマや円マークは不要）
3. **負の値を入力**: 正の値を入力してください

エラーメッセージを確認して、適切な金額を入力してください。

### Q34: 「入金登録」ボタンをクリックしても何も起こりません。

**A**: 以下を確認してください：

1. **ブラウザのJavaScriptが有効か**: JavaScriptが無効になっている可能性があります
2. **ネットワーク接続**: インターネット接続を確認してください
3. **ブラウザのキャッシュ**: ブラウザのキャッシュをクリアしてページをリフレッシュしてください

問題が解決しない場合は、管理者に連絡してください。

### Q35: 入金記録の登録後、請求書のステータスが「支払済み」にならないのはなぜですか？

**A**: 以下の原因が考えられます：

1. **部分入金**: 入金額が請求金額に達していない場合、ステータスは「送付済み」のままです
2. **複数回の部分入金**: 既入金額 + 今回の入金額が請求金額に達していない可能性があります

請求書詳細ページで残額を確認してください。残額が0円になると自動的に「支払済み」になります。

## 権限と制限

### Q36: リーダーは入金記録を登録できますか？

**A**: はい、できます。入金記録機能はリーダー（Leader）以上が利用できます。一般スタッフは入金記録機能にアクセスできません。

### Q37: 一般スタッフは入金履歴を閲覧できますか？

**A**: いいえ、できません。入金記録機能全体（登録・閲覧）がリーダー以上に制限されています。一般スタッフは入金一覧ページや入金履歴にアクセスできません。

### Q38: リーダー、マネージャー、管理者で入金記録の権限に違いはありますか？

**A**: いいえ、入金記録機能に関してはリーダー、マネージャー、管理者の権限は同じです。いずれも入金記録の登録と閲覧ができます。

### Q39: 入金記録を登録できるのは自組織の請求書だけですか？

**A**: はい、自組織（organization_id）の請求書に対してのみ入金記録を登録できます。他組織の請求書は選択肢に表示されません。マルチテナントシステムのため、データは組織ごとに完全に分離されています。

### Q40: 入金記録の登録数に制限はありますか？

**A**: いいえ、入金記録の登録数に制限はありません。同じ請求書に対して何度でも入金記録を登録できます（残額が0円になるまで）。部分入金が多い場合でも問題なく記録できます。

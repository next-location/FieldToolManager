## 基本的な質問

### Q1. 誰が作業報告書を作成できますか？

**A.** **すべての権限**で作業報告書を作成できます。

| 権限 | 作業報告書作成 | 提出 |
|------|--------------|------|
| **Staff** | ✅ | ✅ |
| **Leader** | ✅ | ✅ |
| **Manager** | ✅ | ✅ |
| **Admin** | ✅ | ✅ |

すべてのユーザーが作業報告書を作成・提出できます。

### Q2. 作業報告書作成画面へのアクセス方法は？

**A.** 以下の手順でアクセスできます：

```
1. サイドメニューから「作業報告」をクリック
2. 画面右上の「+ 新規作成」ボタンをクリック
```

または、ダッシュボードから直接「作業報告書作成」のリンクをクリックします。

### Q3. 作業報告書作成に必要な情報は何ですか？

**A.** 以下の情報が必須です：

**必須項目**:
- ✅ 現場
- ✅ 作業日
- ✅ 作業内容

**任意項目**:
- 天気
- 作業時間（開始・終了時刻、休憩時間）
- 時間外（残業時間）
- 作業場所
- 進捗率
- 使用材料
- 使用道具
- 帯同作業員
- 写真
- 特記事項・備考
- カスタムフィールド

### Q4. 作業報告書作成にかかる時間はどのくらいですか？

**A.** 報告書の詳細度によりますが、以下が目安です：

| 報告書の種類 | 所要時間 |
|------------|---------|
| **簡単な報告書**（最小限の入力） | 5分 |
| **通常の報告書**（写真2〜3枚） | 10〜15分 |
| **詳細な報告書**（写真多数、詳細記述） | 20分〜 |

慣れてくると、より短時間で作成できるようになります。

### Q5. 承認は必要ですか？

**A.** **組織の設定により異なります**。

**承認が必要な組織**:
- 提出後、Leader・Adminの承認が必要
- 承認されるまで「提出済み」ステータス
- **注意**: Managerは承認できません

**承認が不要な組織**:
- 提出すると即座に「承認済み」になる
- 上司の承認は不要

組織の設定は、Adminが「作業報告設定」で変更できます。自分の組織の設定がわからない場合は、Adminに確認してください。

## 基本情報入力に関する質問

### Q6. 現場が選択肢に表示されません。どうすればよいですか？

**A.** 以下を確認してください：

**1. 現場が登録されているか確認**
- 現場管理で現場を登録してください

**2. 現場がアクティブか確認**
- 「is_active」がtrueの現場のみ選択可能
- 非アクティブ化された現場は表示されません

**3. 現場が削除されていないか確認**
- 削除された現場は表示されません

**4. Adminに連絡**
- 上記を確認しても表示されない場合は、Adminに連絡してください

### Q7. 過去の日付で作業報告書を作成できますか？

**A.** はい、**過去の日付で作成できます**。

**用途**:
- 後からの報告
- 報告し忘れた作業の記録
- 休暇後の報告

**方法**:
1. 「作業日」フィールドでカレンダーを開く
2. 過去の日付を選択
3. 通常通り報告書を作成

**注意**: 未来の日付も選択できますが、通常は使用しません。

### Q8. 天気は必須ですか？

**A.** いいえ、天気は**任意項目**です。選択しなくても作業報告書を作成できます。

**天気を記録する理由**:
- 天候による作業への影響を記録
- 雨天による作業遅延の記録
- 安全管理の記録

通常は天気を選択することを推奨しますが、必須ではありません。

### Q9. 天気の絵文字が表示されないのですが

**A.** 一部のブラウザや環境では、絵文字が正しく表示されない場合があります。

**対応方法**:
- ☀️ 晴れ → 「晴れ」を選択
- ☁️ 曇り → 「曇り」を選択
- 🌧️ 雨 → 「雨」を選択
- ⛄ 雪 → 「雪」を選択

絵文字が表示されなくても、選択肢のテキストで判断できます。

### Q10. 現場の住所が表示されないのですが

**A.** 現場に住所が登録されていない場合、住所は表示されません。

**表示例**:
```
住所あり: A現場（東京都新宿区〇〇）
住所なし: A現場
```

現場名だけで判断するか、住所が必要な場合はAdminに登録を依頼してください。

## 作業時間に関する質問

### Q11. 作業時間のデフォルト値は変更できますか？

**A.** いいえ、現在のバージョンでは**デフォルト値は固定**されています。

**デフォルト値**:
- 開始時刻: 08:00
- 終了時刻: 17:00
- 休憩時間: 60分

毎回、実際の作業時間に合わせて変更してください。

### Q12. 実作業時間が正しく計算されません

**A.** 以下を確認してください：

**1. 開始・終了時刻を正しく入力しているか確認**
```
例:
開始時刻: 08:00
終了時刻: 17:00
休憩時間: 60分
↓
実作業時間: 8.0時間（正しい）
```

**2. 終了時刻が開始時刻より後か確認**
- 終了時刻が開始時刻より前の場合、計算が正しくできません

**3. 休憩時間が多すぎないか確認**
- 休憩時間が作業時間を超えると、実作業時間が0または負になります

### Q13. 時間外（残業時間）と実作業時間の違いは何ですか？

**A.** 以下の違いがあります：

**実作業時間**:
- 開始時刻から終了時刻までの作業時間（休憩時間を除く）
- 自動計算
- 通常の勤務時間を含む

**時間外（残業時間）**:
- 通常の勤務時間を超えた時間
- 手動入力
- 残業手当の計算に使用

**例**:
```
開始 08:00、終了 20:00、休憩 60分
→ 実作業時間: 11.0時間

通常の勤務時間: 8時間
残業時間: 3時間（11 - 8 = 3）
→ 時間外: 3 時間
```

### Q14. 休憩時間を複数回に分けて取った場合はどうすればよいですか？

**A.** **合計の休憩時間を入力**してください。

**例**:
```
昼休憩: 60分
午後の休憩: 15分
合計: 75分
↓
休憩時間: 75 を入力
```

### Q15. 作業時間を15分刻みで入力できますか？

**A.** はい、時刻選択で15分刻みの入力が可能です（ブラウザにより異なります）。

**例**:
```
08:00
08:15
08:30
08:45
```

ただし、実作業時間は0.1時間単位（6分単位）で計算されます。

## 作業内容に関する質問

### Q16. 作業内容はどのように書けばよいですか？

**A.** 以下のポイントを押さえて記述してください：

**推奨の書き方**:
- ✅ 箇条書き（・または-で始める）
- ✅ 具体的な作業内容
- ✅ 作業箇所の明記
- ✅ 完了・未完了の明記

**例**:
```
✅ 良い例:
・基礎型枠の組立（1F部分）完了
・コンクリート打設準備
・鉄筋の配筋確認（検査合格）
・安全柵の設置（東側）

❌ 悪い例:
・作業しました
・いつも通り
・型枠
```

### Q17. 作業内容の文字数制限はありますか？

**A.** 明確な文字数制限はありませんが、**適度な長さ**で記載することを推奨します。

**推奨**:
- 200〜500文字程度
- 5〜10行の箇条書き

長すぎる場合は、「特記事項」に詳細を記載することを検討してください。

### Q18. 進捗率はどのように入力すればよいですか？

**A.** **作業全体に対する完了率**を0〜100の範囲で入力してください。

**入力例**:
```
初日: 10（10%完了）
2日目: 30（30%完了）
3日目: 50（50%完了）
...
最終日: 100（100%完了）
```

**注意**: 進捗率は任意項目のため、入力しなくても問題ありません。

### Q19. 使用材料と使用道具の違いは何ですか？

**A.** 以下の違いがあります：

| 項目 | 使用材料 | 使用道具 |
|------|---------|---------|
| **対象** | 消耗品、材料 | 道具、機械 |
| **例** | コンクリート、鉄筋、木材 | 電動ドリル、エアコンプレッサー |
| **数量** | 使用した数量を記載 | 使用した道具を列挙 |
| **記録方法** | テキストで記述 | 道具マスタから選択またはテキスト |

### Q20. 道具マスタから選択する利点は何ですか？

**A.** 以下の利点があります：

**利点**:
- ✅ 入力が簡単（選択するだけ）
- ✅ 表記揺れがない（統一された名称）
- ✅ 道具の使用履歴を記録できる
- ✅ 道具の稼働率を分析できる

**デメリット**:
- 道具マスタに登録されていない道具は選択できない

道具マスタに登録されていない道具は、テキストエリアに記述してください。

## 帯同作業員に関する質問

### Q21. 帯同作業員は必須ですか？

**A.** いいえ、帯同作業員は**任意項目**です。一人で作業した場合は、登録しなくても問題ありません。

**登録する場合**:
- 複数人で作業した場合
- 作業員ごとの残業時間を記録したい場合

### Q22. 自分以外の作業員を登録できますか？

**A.** はい、**同じ組織のユーザー**であれば登録できます。

**登録できるユーザー**:
- 同じ組織のStaff
- 同じ組織のLeader
- 同じ組織のManager
- 同じ組織のAdmin

**登録できないユーザー**:
- 削除されたユーザー
- 別の組織のユーザー

### Q23. 帯同作業員を検索できますか？

**A.** はい、**検索ボックスで作業員を検索**できます。

**検索方法**:
1. 作業員選択のドロップダウンをクリック
2. 検索ボックスに作業員名を入力
3. 候補が絞り込まれる
4. 作業員を選択

**検索例**:
```
入力: 「山田」
↓
候補:
- 山田太郎
- 山田花子
```

### Q24. 帯同作業員の実作業時間は自動的に設定されますか？

**A.** はい、帯同作業員の実作業時間は**作成者と同じ時間が自動的に適用**されます。

**例**:
```
作成者:
- 開始 08:00、終了 17:00、休憩 60分
- 実作業時間: 8.0時間

帯同作業員（山田太郎）:
- 実作業時間: 8.0時間（自動適用）
```

**注意**: 帯同作業員の実作業時間を個別に設定することはできません。異なる作業時間の場合は、別々の作業報告書を作成してください。

### Q25. 帯同作業員の残業時間はどうやって入力しますか？

**A.** 各作業員の「時間外」フィールドに残業時間を入力してください。

**入力方法**:
```
帯同作業員:
┌────────────────────┐
│ 山田太郎            │
│ 時間外: [2] 時間   │ ← ここに入力
└────────────────────┘
```

作業員ごとに異なる残業時間を設定できます。

## 写真添付に関する質問

### Q26. 写真は必須ですか？

**A.** いいえ、写真は**任意項目**です。写真を添付しなくても作業報告書を作成できます。

**写真を添付する利点**:
- ✅ 作業の様子を視覚的に記録
- ✅ 作業進捗の証拠
- ✅ 安全確認の記録
- ✅ 顧客への報告資料

### Q27. 写真は何枚まで添付できますか？

**A.** 写真の枚数に明確な制限はありませんが、**5〜10枚程度**を推奨します。

**推奨枚数**:
- 簡単な作業: 2〜3枚
- 通常の作業: 5枚程度
- 重要な作業: 10枚程度

多すぎる場合は、重要な写真に絞ることを検討してください。

### Q28. 対応していないファイル形式のエラーが表示されます

**A.** 以下のファイル形式のみ対応しています：

**対応形式**:
- JPEG（.jpg, .jpeg）
- PNG（.png）
- WebP（.webp）

**非対応形式**:
- GIF（.gif）
- BMP（.bmp）
- TIFF（.tiff）

非対応形式の場合は、画像編集ソフトでJPEGまたはPNGに変換してください。

### Q29. 「ファイルサイズが大きすぎます」というエラーが表示されます

**A.** 写真のファイルサイズが**5MBを超えています**。

**対処法**:

**方法1: 写真を圧縮**
- 画像編集ソフトやオンラインツールで圧縮
- 推奨サイズ: 1MB以下

**方法2: 画質を下げる**
- スマートフォンのカメラ設定で画質を下げる
- 「標準」または「低」画質を使用

**方法3: リサイズ**
- 画像の解像度を下げる
- 推奨解像度: 1920×1080以下

### Q30. 写真の順番を変更できますか？

**A.** はい、**PCではドラッグ&ドロップで順番を変更**できます。

**変更方法（PC）**:
1. 写真をマウスでドラッグ
2. 希望の位置にドロップ
3. 順番が変更される

**スマートフォン**:
- 現在のバージョンでは、アップロード順で固定されます
- 順番を変更したい場合は、アップロード順を調整してください

### Q31. キャプション（説明）は必須ですか？

**A.** いいえ、キャプションは**任意項目**です。ただし、**入力を推奨**します。

**キャプションを入力する利点**:
- 写真の内容が明確になる
- 後から見返したときに分かりやすい
- 上司や顧客への説明が容易

**入力例**:
```
・基礎コンクリート打設前
・配筋検査合格
・1階柱施工完了
```

## 保存・提出に関する質問

### Q32. 「下書き保存」と「提出する」の違いは何ですか？

**A.** 以下の違いがあります：

| 項目 | 下書き保存 | 提出する |
|------|-----------|---------|
| **ステータス** | 下書き（灰色） | 提出済み（オレンジ色） |
| **編集** | 可能 | 不可 |
| **承認** | 不要 | 必要（設定による） |
| **用途** | 途中保存、後で修正 | 上司への報告 |

**使い分け**:
- 途中まで作成: 「下書き保存」
- 内容が確定: 「提出する」

### Q33. 下書き保存した作業報告書は後で編集できますか？

**A.** はい、**下書き保存した作業報告書は何度でも編集できます**。

**編集方法**:
1. 作業報告一覧から作業報告書をクリック
2. 作業報告書詳細画面を開く
3. 「編集」ボタンをクリック
4. 作業報告書編集画面で修正
5. 「下書き保存」または「提出する」をクリック

### Q34. 提出後、間違いに気づきました。どうすればよいですか？

**A.** 以下の対応を取ってください：

**ステップ1: 上司（Manager/Admin）に連絡**
- すぐに上司に連絡
- 「作業報告書に誤りがあります」と伝える
- 修正内容を説明

**ステップ2: 上司が差し戻し**
- 上司が「差し戻し」を実行
- ステータスが「下書き」に戻る

**ステップ3: 修正して再提出**
- 「編集」ボタンをクリック
- 誤りを修正
- 「提出する」をクリック

**予防策**: 提出前に内容を十分に確認してください。

### Q35. 提出しないで、下書きのままにしておくことはできますか？

**A.** はい、**下書きのまま保存しておくことができます**。

**用途**:
- 後で写真を追加する予定
- 上司に口頭で確認してから提出
- 作業中の報告書

**注意**: 下書きのままでは承認されないため、最終的には必ず提出してください。

## トラブルシューティング

### Q36. 「必須項目を入力してください」というエラーが表示されます

**A.** 以下の必須項目が未入力の可能性があります：

**確認項目**:
- ✅ 現場を選択したか
- ✅ 作業日を入力したか
- ✅ 作業内容を入力したか
- ✅ カスタムフィールドの必須項目を入力したか（*マーク）

すべての必須項目を入力してから、再度保存・提出してください。

### Q37. 「作業報告書の作成に失敗しました」というエラーが表示されます

**A.** 以下を確認してください：

**1. インターネット接続を確認**
- Wi-Fiまたはモバイルデータ通信が接続されているか確認

**2. ブラウザのキャッシュをクリア**
- ブラウザのキャッシュをクリアして再試行

**3. 写真のファイルサイズを確認**
- 写真が5MBを超えていないか確認
- 大きい場合は圧縮

**4. 管理者に連絡**
- 上記を試しても解決しない場合は、管理者に連絡してください

### Q38. 写真がアップロードできません

**A.** 以下を確認してください：

**1. ファイル形式を確認**
- JPEG、PNG、WebPのいずれかか確認

**2. ファイルサイズを確認**
- 5MB以下か確認

**3. インターネット接続を確認**
- 接続が安定しているか確認

**4. ブラウザを確認**
- 別のブラウザ（Chrome、Firefox、Edge）で試してください

### Q39. 保存ボタンをクリックしても反応がありません

**A.** 以下を確認してください：

**1. 必須項目を入力しているか確認**
- 未入力の必須項目があると、保存できません
- エラーメッセージが表示されていないか確認

**2. ボタンをダブルクリックしていないか確認**
- ボタンは1回だけクリックしてください
- 処理中は「保存中...」と表示されます

**3. ブラウザのJavaScriptを確認**
- JavaScriptが有効になっているか確認

**4. ページを再読み込み**
- 一時的なエラーの可能性
- ページを再読み込みして再試行（入力内容は失われます）

### Q40. カスタムフィールドが表示されません

**A.** 以下を確認してください：

**1. 組織でカスタムフィールドが設定されているか確認**
- カスタムフィールドが設定されていない場合は表示されません
- Adminに確認してください

**2. 現場を選択しているか確認**
- 現場固有のカスタムフィールドは、現場を選択後に表示されます

**3. ページを再読み込み**
- 一時的なエラーの可能性
- ページを再読み込みして再試行
